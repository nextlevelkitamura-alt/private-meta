custom-agent-creator

Claude Code / Codex / OpenCode 向けのカスタムエージェント定義を、目的・権限・モデル・保存先・呼び出し方に応じて設計・整理・レビューする支援Skill。Skill自体はエージェントではなく、定義ファイルを作るための作成支援。

いつ使うか（発火する発話例）

  • 「Claude Code の subagent を作りたい」「quality-gate を作って」
  • 「Codex の reviewer / custom agent を作って」
  • 「OpenCode の quality-gate agent を設計して」
  • 「permission / sandbox / approval を設計したい」「inline MCP を持たせたい」
  • 「codex-reviewer を作りたい」「hook-designer を作りたい」

※ Skillそのものの作成・改善は skill-creator-custom / skill-creator-codex。エージェントの運用・指揮は cockpit-supervisor、作業の入口判断は plan-triage。これらには使わない。

構造

workflows/ は持たない（作業が単一フロー: ツール判定→該当reference→設計・生成→checklist→出力）。読み分けは SKILL.md が担い、対象ツールの reference だけを読む。

絶対ルール・安全方針

  • 副作用は定義ファイルの作成・編集のみ。エージェントの実行はしない。
  • reviewer / evaluator / quality-gate は原則 read-only（Claude=編集ツールを渡さない、Codex=sandbox_mode: read-only、OpenCode=edit: deny）。
  • secret / token を定義に書かない。bypassPermissions / danger-full-access / --auto 前提や destructive command の無条件許可を禁止。
  • 保存先の global / project を明示してから書く。model ID・キー名はバージョンで変わるため実装時に公式確認（特に OpenCode Go の model ID）。
  • 言語: 自然言語は日本語、frontmatter key / enum / model ID / path / command / agent名 / Skill名 は英語ASCII固定。
