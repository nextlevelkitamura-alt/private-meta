# light.md — ライトモード（4ラリー）

社内共有・個人資料・速報など、サクッと作りたいときの4ラリーフロー。
**重要度"高"（営業・提案・外部公開）の場合のみ追加3問の深掘りを差し込む。**

## 全体像

```
ラリー1: 内容（誰に・何を・何のために）+ 重要度判定
ラリー2: Claude が4要素を一括提案 → 各要素を A/B/C で選択
ラリー3: カスタム選択分の個別ヒアリング + 情報源
ラリー4: 構成案提示 → 画像どうする → エンジン選択 → 実行
```

---

## ラリー1: 内容ヒアリング

### 必須3問（1メッセージで一括）

```
ライトで進めますね。3つだけ教えてください。

1. 誰に向けた資料？（聴衆・1人〜複数）
2. 何を伝える？（テーマ・トピック）
3. 何のため？（見終わった人にどんな行動を取ってほしい？）
```

### 重要度判定（Claudeが裏で判定）

回答を見て次の判定:
- **重要度"高"**: 営業 / 提案 / 社外提出 / 採用 / 講演 / 経営報告
- **重要度"標準"**: 社内共有 / メモ / 進捗報告 / 個人用

### 重要度"高"のときだけ追加3問

```
重要そうなので、もう3つだけ:
4. 相手の最大の懸念・反論は？
5. 決裁プロセス・キーパーソンは誰？
6. 数値根拠（データ・実績）はある？
```

→ **これは7問ロジックツリー Q3（Villain/Stakes）・Q4（決裁構造）・Q5の根拠 に対応。**

---

## ラリー2: 4要素一括提案

回答を踏まえて、Claude が4要素を**理由つきで**一括提案する。

### 提案テンプレ

```
内容を踏まえて、以下を提案します:

┌─────────────────────────────────────────────┐
│ ① 差別化視点 → {推奨フレーム1〜2個}
│   理由: {なぜこのフレームが効くか}
│
│ ② 構成 → {推奨構成テンプレ名}・{推奨枚数}枚
│   理由: {なぜこの構成・枚数か}
│
│ ③ レイアウト → {推奨レイアウト}（全体 or スライドごと）
│   理由: {なぜこのレイアウトか}
│
│ ④ ブランド → {推奨ブランド名}
│   理由: {_last-used.txt or 内容との適合}
└─────────────────────────────────────────────┘

各要素について選んでください:
[A] このまま採用
[B] カスタムで作る（ラリー3で個別ヒアリング）
[C] 不要・使わない

例: ①A ②A ③B ④A
```

### 推奨ロジック

#### 構成テンプレの選び方

| 内容パターン | 推奨構成 |
|---|---|
| 営業・顧客提案 | `営業提案` |
| 短時間決裁・経営報告 | `圧縮` |
| 講演・登壇 | `講演登壇` |
| 自己紹介・個人ブランディング | `自己紹介` |
| 社内進捗・ナレッジ展開 | `社内共有` |
| 商品・サービス紹介（短時間） | `ベネフィット` |
| 複雑な課題への解決提案 | `PASTOR` |

#### 枚数の推奨ロジック

```
提示時間 → 枚数の目安:
- 3分以内 → 5-7枚
- 5分 → 7-9枚
- 10分 → 10-14枚
- 15-20分 → 15-20枚
- 30分以上 → 20-30枚
- 配布のみ（時間制約なし） → 内容次第・15-25枚
```

構成テンプレの `recommended_slides` と突き合わせて最終決定。

#### 差別化フレームの選び方

| 内容パターン | 推奨フレーム |
|---|---|
| 営業・提案 | 体験ストーリー + スタンス表明 |
| 講演・登壇 | 独自視点 + STAR Moment（体験ストーリー or アンチテーゼ） |
| 自己紹介 | 体験ストーリー + 独自視点 |
| 経営報告・分析 | 数値の独自解釈 |
| 思想発信 | アンチテーゼ + 独自視点 |
| 社内共有（重要度標準） | スキップ可 |

#### レイアウトの選び方

内容次第。デフォルトは複数レイアウトを混在（章扉=全画面写真、本論=右画像-左文字、データ=データ表-グラフ大、キーメッセージ=ミニマル文字のみ など）。

#### ブランドの選び方

1. `テンプレ/ブランド/_last-used.txt` を読む
2. その名前を第一推奨に
3. 内容と合わなそうなら他のブランドを推奨

## ラリー3: カスタム分の個別ヒアリング + 情報源

### カスタム要素の対話（B選択分のみ・順番に）

#### ① 差別化をカスタム
```
差別化はどう作りますか？
- どの軸（独自視点 / 体験 / スタンス / 数値解釈 / アンチテーゼ）が合いそう？
- それとも全く別の切り口？
```
詳細は `テンプレ/差別化/{選択した軸}.md` を読み、核心の問いを投げる。

#### ② 構成をカスタム
```
構成はどう組みますか？
- 章立てを自由に作る場合、何枚で・どんな流れにしたい？
- 既存テンプレ（PASTOR/ベネフィット/...）の中で改変したいものは？
```

#### ③ レイアウトをカスタム
```
レイアウトの希望は？
- 全体トーン（ミニマル / 標準 / 高密度）
- 図解 vs 文字 vs 画像のバランス
- 雰囲気のリファレンス（過去資料・サイト・キーワード）
- NGイメージ
```

#### ④ ブランドをカスタム / 新規
新規作成なら `scripts/register-brand.ts` を案内、または対話で:
```
新規ブランドを作ります。3問だけ:
1. ロゴ画像のパス（任意・後で追加可）
2. メインカラー（HEX or 色名）
3. トーン（プレミアム / 親しみ / クール / 温かみ など）
```
→ `テンプレ/ブランド/{name}/brand.yaml` を生成、ロゴがあれば同ディレクトリにコピー。

### 情報源の選択（必ず聞く）

```
情報源はどうしますか？（複数選択可）
A) ~/Private/副業/素材/wiki/ から引く（ローカル既存ナレッジ活用）
B) ネット情報をリサーチ官に集めさせる
C) 添付資料あり（パスを教えてください）
D) 追加で聞き出してほしいことがある（こちらから質問しますか？）
E) 今ある情報で十分
```

- A → 素材/wiki/ を grep/Read。テーマに合うアトミックノートを抽出して素材として渡す
  - 例: `grep -r "登壇" ~/Private/副業/素材/wiki/`
  - `usable_in: [登壇]` のfrontmatter を持つノートを優先
- B → リサーチ官 Agent を呼ぶ（`references/リサーチ官-prompt.md`）
- C → ファイル読み込み・要約
- D → Claude が必要そうな追加質問を3-5問
- E → そのまま次へ

## ラリー4: 構成案 → 画像 → エンジン → 実行

### 4-a. スライドマップ提示

```yaml
storyline: {構成テンプレ名}
total_slides: {枚数}
chapters:
  - "[1] {章タイトル} ({枚数}枚・{テキスト量タグ}・{推奨レイアウト})"
  ...
key_message: "{13-30字の一言}"
differentiation: "{差別化フレームから抽出した一文}"
brand: "{ブランド名}"
```

提示して「これでOK？修正点ある？」と確認。

### 4-b. 画像どうする？（必ず聞く）

```
画像はどうしますか？（必要なスライドが {N} 枚あります）
A) Codex image_gen で生成する
B) 画像生成APIを直叩き（gpt-image / Imagen / Gemini）
C) 添付済みの画像を使う
D) 画像なしで進める
```

A選択時は `/generate-images` を起動し、Codex `image_gen` に各スライド分のプロンプトを渡す。

### 4-c. エンジン選択（必ず聞く）

```
生成エンジンを選んでください:
A) NotebookLM（ソース忠実・社内向き）
B) Claude for PowerPoint アドイン（ブランド踏襲・外部向き）
```

- A → `references/notebooklm-route.md`
- B → `references/powerpoint-route.md`

### 4-d. brief.yaml 保存 + 実行

```bash
mkdir -p ~/.claude/skills/slide/ログ/$(date +%Y-%m)-{資料名}/
cp ~/.claude/skills/slide/ログ/brief-template.yaml \
   ~/.claude/skills/slide/ログ/$(date +%Y-%m)-{資料名}/brief.yaml
# Edit で値を埋める
```

→ エンジンルートに従って生成。

### 4-e. 後処理 + Archive

- ブランド使用: `テンプレ/ブランド/_last-used.txt` を今回のブランド名で上書き
- 完了後: `ログ/{資料名}/review.md` に評価・気づきを記録
- **Archive**: 完成 .pptx を `~/Private/副業/素材/decks/{資料名}.pptx` にコピー
- **Archive (素材逆流)**: 強い素材（キャッチコピー・図解構造・検証済み数字・体験エピソード）を `~/Private/副業/素材/wiki/{カテゴリ}/` に書き戻し
  - 必ずユーザーに「これを wiki に保存しますか？」と確認
  - frontmatter テンプレは `~/Private/副業/素材/README.md` 参照

## ライトの所要時間目安

- 重要度"標準" + 全A選択: 5-10分
- 重要度"高" + 一部カスタム: 15-25分
- 全部カスタム + リサーチ官あり: 30-45分（しっかりに近い）

→ 全部カスタムが多くなるなら最初から **しっかり (full.md)** を選ぶよう案内する。
