skill-creator-codex

OpenAI/Codex公式の skill-creator を日本語運用向けに改編し、Codex向けSkillの新規作成・既存Skill更新・agents/openai.yaml 生成・quick_validate 検証を支援する。Skillライフサイクルの窓口 skill-creator-custom から委譲されたCodex固有作業を担う。

いつ使うか（発火する発話例）

  • 「Codexで使うSkillを作りたい」
  • 「SKILL.mdを更新したい」
  • 「scripts / references / assets を整理したい」
  • 「Skillの発火条件やメタデータ（agents/openai.yaml）を改善したい」

Skillライフサイクル判断（作る/直すの境界、Global/repo-local、削除・移行）はここではなく skill-creator-custom が担う。迷ったらそちらへ戻す。

構造

referencesの中身（何が書いてあるか）

openai_yaml.md — agents/openai.yaml のフィールド仕様

  • Full example: interface（display_name / short_description / icon_small / icon_large / brand_color / default_prompt）と dependencies.tools（mcp依存の記法）の全体像
  • Field descriptions and constraints: 各フィールドの意味と制約（文字列は必ずquote、short_descriptionは25-64文字、default_promptは$skill-name形式で対象Skillを明示 等）

いつ読む: agents/openai.yaml を生成・更新する前に必ず読む（scripts/generate_openai_yaml.py 実行前後の突合含む）。

絶対ルール・安全方針

  1. Skill初期化先の区別（2026-07-18更新）: ~/.codex/skills（=$CODEX_HOME/skills）はCodexが自動で読む一時置き場（scratch）。Codex単体で使い捨てる試作だけをここに置く。
  2. 北村環境でGlobal Skillとして長期運用する場合は ~/.codex/skills に残さない。正本は AIエージェント基盤/skills/<skill-name>/ に置き、global-skill-registry/scripts/link-global-skill.sh で ~/.agents/skills（Codexが読む）・~/.claude/skills・~/.gemini/... の4窓へdirect symlink露出する。~/.codex/skills は露出先にしない。runtime入口を正本にしない。
  3. フォルダは SKILL.md ＋ agents/ scripts/ references/ assets/ 以外を作らない（SKILL.htmlは唯一の人間向け例外ファイル）。README・INSTALLATION_GUIDE・CHANGELOGなどSkill実行に直接必要ない補助文書は原則作らない。
  4. 削除・正本移動・symlink張り替え・外部送信・破壊的操作は人間の明示承認が必須。repo-local Skillを無理にGlobal化しない。
  5. scriptを追加・変更したら代表ケースで実行確認する。placeholderや不要なexampleは残さない。
  6. 作成・更新後は scripts/quick_validate.py を実行し、frontmatter有効性・nameとフォルダ名一致・descriptionの発火十分性・agents/openai.yamlとの整合・SKILL.mdの肥大化なしを確認する。
  7. Skill編集の完了条件は SKILL.html の再生成まで（このSkill自身にも適用）。
