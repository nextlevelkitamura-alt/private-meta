# 05 段階移行と安全化

## 目的

Focusmap repo-local runtimeを安全化し、現行focusmap-agentとlegacy task-runnerを段階的に一本化する。Global board-sweep / session-board / Turso parityは子09へ分離し、本子計画から触らない。

## 現状

### 最優先

- Focusmap Stop hook 2本が `git add -A` で全変更を自動commitする。
- push guardが現行AGENTSと矛盾する。
- `.claude/settings.local.json` に旧pathと広いpermissionが残る。

### runtimeドリフト

- 実機は旧 `com.focusmap.codex-app-server` だけloaded。
- official focusmap-agent / official app-server / task-runnerはnot-loaded。
- 現行focusmap-agentはheartbeat / claim / command / Codex monitorを持つが、processは動いていない。
- 旧task-runnerは巨大な責務と再導入導線が残る。

## 方針と順序

### A. repo-local hook安全化

1. auto-commitを一旦停止候補とし、人間判断で「廃止」または「明示されたpathだけcommit」にする。
2. push guardを現行AGENTSと一致させる。明示pushを誤って拒否しない。
3. settings.localの旧path / permissionは、利用実態を確認してから縮小する。無断削除しない。

### B. Focusmap agent一本化

既存 `codex-mac-agent-unification` と未追跡 `task-runner-retirement-agent-single-path` を再利用する。新しい退役仕様を本計画へコピーしない。

1. 旧runnerにだけ残るschedule、repo scan、staff-status、recurrence / package / Claude RC等を列挙する。
2. 現行agentへ移す / 廃止 / 人間運用に残すを1件ずつ裁定する。
3. installer / setup / Mac supervisorから旧runnerの通常起動導線を外す。
4. 人間ゲートで意図したofficial labelだけをloadし、旧labelを停止する。
5. registry / runtime / processが一致したことをverifyする。

### C. 観察と縮退

最低1週間、read-only UIとログで欠測・重複・staleを観察する。旧alias / route / registered診断は観察中残し、削除は別承認にする。

## 完了条件（レビュー項目）

- [ ] Focusmap `.claude` / `.codex` Stop hookに `git add -A` がなく、他agent / 他作業差分をcommitしない。
- [ ] push guardとpermissionが現行Focusmap AGENTSに一致し、旧絶対pathが現役設定に残らない。
- [ ] 旧task-runnerにしかない責務が全件裁定され、通常起動導線が現行agentへ一本化される。
- [ ] 意図したlaunchd label / process / entrypointと実機が一致する。
- [ ] rollback先が危険auto-commit無効・旧runner停止・既存Web継続のread-only baselineであり、危険な旧状態を復活させない。
- [ ] 1週間の観察でrepo-local hookの誤commitとagent二重起動が0件である。
