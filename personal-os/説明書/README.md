# パーソナルOS 説明書（現在の運用モデル）

会話をまたいでも「**今どう使うか**」が分かるように残す、パーソナルOSの入口ドキュメント。
運用モデルが変わったらここを更新する。パスは特記なき限り **personal-os 直下基準**。最終更新: 2026-07-02。

## 1. パーソナルOSとは

AIが半自動であなたの仕事を回す仕組み。実装repoではなく、AI作業を迷わず進める運用の中枢。
入口ルールの正本は `AGENTS.md`（personal-os 直下）。この説明書は「運用モデルの現在地」を残す補助。

## 2. 今の運用モデル：コックピット方式

- Orca 上に「**計画+監督 / 実装 / レビュー**」の3ペインを立て、**あなた＝指揮官が直接AIを操縦**する。
- 立てる: `orca-cockpit` スキル → `cockpit.sh up --repo <r> --branch <b> --title "<計画名>"`
  （既定=計画claude・実装codex・レビューcodex ／ 並列起動 約9秒 ／ codex更新は自動Skip＋4つ目の端末で自動update）。撤去は `cockpit.sh down`。
- 正本: `AIエージェント基盤/skills/orca-cockpit/`（`SKILL.md` ＋ `scripts/cockpit.sh`）。

## 3. 状態の持ち方：2モード（契約 §1・2026-07-02 裁定）

- **既定＝モードB（テキスト状態・単一指揮官）**: あなた＝コックピットが直接配る間は、テキストのリストで状態を持つ。
  → **ai-jobs のフォルダ受け渡し（`ready→…→archive` の mv）は今は使わない＝保留**。
- **モードA（フォルダmv）**: 無人・大勢のAIが並行で自分から取りに来る時だけ、ai-jobs の5フォルダの移動で状態管理。B で安定したものを卒業させる先。
- 不変: どちらでも「状態は1箇所」（二重管理禁止）。
- 契約正本: `説明書/運用契約.md`（2026-07-02 昇格。段階語彙・モードA活性化条件・破壊操作ゲートも同契約）。

## 4. なぜこの形か（固定概念の更新）

- 以前の前提は「AIの仕事は全部 ai-jobs キューに乗せるのが正本」（＝無人・並行前提）。
- **orca-cockpit の完成で「生きた指揮官が数体を直接操縦」が現実**になり、今はキューは過剰＝B で十分、に更新された（2026-07-02）。
- つまりこの変更は「道具が増えて前提が更新された」もの。ai-jobs は捨てず、モードA として将来用に温存。
