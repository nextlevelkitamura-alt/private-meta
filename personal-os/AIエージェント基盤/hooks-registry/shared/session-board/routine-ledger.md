# routine-ledger — 定型作業の判定台帳（board-sweep 用）

ボードの⏸（停止・確認待ち）行のうち「定型作業」を、人間確認なしで「終わったこと」へ流してよいか
判定するための台帳。読むのは `loops-registry/loops/board-sweep/scripts/sweep.py`（読み取り専用・
パース失敗は無害＝全行 unknown）。台帳へのエントリ追加・確定（ドラフト解除）は、候補、根拠、重複、readbackを確認してAIが行う。

## 書式（1定型=1節・キー5つ）

- `一致:` ⏸行とのマッチ条件。`;` 区切りの AND。値内の `|` は OR（いずれか一致）。
  使えるキー: `repo=`（basename一致）／`who=`（runtime または runtime/model の前方一致）／
  `goal前方一致=`／`goal含む=`／`now前方一致=`／`実体=なし|あり`（実体transcriptの有無）。
- `終わり:` 何をもって「実際に終わった」と見なすか（人間向けの判断基準1行）。
- `確認:` 読み取り専用の実体確認。`none`（確認不能・一致のみで判定）／`file:<パス>`（存在=OK）／
  `log:<パス>`（存在=OK）／`cmd:<コマンド>`（exit 0=OK・**読み取り専用のコマンドのみ書く**）。
- `記載:` 自動finish時に「終わったこと」へ書く子entryの文言（`[auto]` プレフィックスと根拠は sweep が自動付与）。
- `判定:` `done`=一致＋確認OKで自動finish可／`unknown`=流し込まない（dry-runログのみ）。
- 節内に「ドラフト」の語がある間は**自動finishされない**（dry-runログにのみ出る）。
  人間が内容を確認したらドラフト行を消して確定する。

## 架電

- 一致: repo=仕事; goal前方一致=朝架電
- 終わり: 朝架電リストのJ列更新（GAS反映）が完了し、セッションが用済みで沈黙している
- 確認: none
- 記載: 朝架電J列の定型更新を完了
- 判定: done
- ドラフト・人間確認待ち（2026-07-09 起票・子05 dry-run版）

## 印刷

- 一致: repo=仕事; goal含む=印刷更新
- 終わり: 架電リスト等の印刷ジョブ送信が完了し、セッションが用済みで沈黙している
- 確認: none
- 記載: 印刷更新の定型作業を完了
- 判定: done
- ドラフト・人間確認待ち（2026-07-09 起票・子05 dry-run版。`goal含む` は「関東印刷更新」等の
  接頭語付きも拾う意図。前方一致に絞るかは人間判断）

## focusmap定期

- 一致: who=codex; now前方一致=# Overview Generate|You are an expert at uph; 実体=なし
- 終わり: focusmapのリモートスレッドはローカルに実体transcriptを持たない（ボードへの誤登録・掃除対象）
- 確認: none
- 記載: focusmapリモートスレッドの誤登録行を掃除
- 判定: done
- ドラフト・人間確認待ち（2026-07-09 起票・子05 dry-run版。goal・repo が `?` のため
  finish の親は「今」列の文言・repo見出しは `### ?` になる点を dry-run で確認する）

## 解除前の必須修正（敵対的レビュー 2026-07-09・全エントリのドラフト解除ゲート条件）

- 一致条件に負条件キー（例: `goal除外=改修|整理|調査|計画`）を導入し、「朝架電J列**の再発防止整理**」型の
  改修・調査セッション誤マッチを塞ぐ（sweep.py 側のマッチ実装込み。実データdry-runで誤マッチ実証済み）。
- 実体なし行が沈黙ガードを経ずに即適格となる経路（sweep.py の実体None時skip）を修正する。
- エントリ名は業務カテゴリ名で運用する（架電・印刷・focusmap定期。2026-07-09人間要望・リネーム済み）。
  「経理」カテゴリの新設は一致条件を人間と確定してから（現状は経理系one-shotはtask_complete経路で適格化）。
