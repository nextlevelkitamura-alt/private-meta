## 2. 今の運用モデル：コックピット方式（既定2ペイン・2026-07-02裁定）

- Orca 上に「**実装 / レビュー**」の2ペインを1レーンとして立て、配布・監視・差し戻し・マージは**指揮官チャット＋見張り番（watch.sh）**が担う。計画が未成熟な子に限り3ペイン目（計画+監督）を構成カードで宣言して追加。
- 立てる: `orca-cockpit` スキル → `cockpit.sh plan`（構成カード提示）→ 人間OK → `cockpit.sh up`（既定=実装claude sonnet5・レビューcodex xhigh）。撤去は `cockpit.sh down`（人間承認後）。
- 正本: 機構=`AIエージェント基盤/skills/orca-cockpit/`、指揮官の判断手順=`AIエージェント基盤/skills/cockpit-supervisor/SKILL.md`。
- `~/orca/workspaces/` は Orca が自動管理する**使い捨ての作業机置き場**（git worktree。履歴の本棚=.gitは正本repoと共有なので、机上のコミットは即・正本側の履歴に記録される。マージ後に `down` で机ごと削除して良い）。`~/.claude` と同類のツールruntime領域であり、正本ではないので `personal-os/` 配下には移さない（2026-07-03裁定）。
