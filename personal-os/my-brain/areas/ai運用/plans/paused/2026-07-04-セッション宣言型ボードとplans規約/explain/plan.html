セッション宣言型ボードとplans規約

デイリーは「動いているエージェント」「終わったこと」の2節だけ。ボードには今生きているセッションだけが並び、完了は人間確認を経て行ごと「終わったこと」へ移動。確実性はhooks/正本のスクリプトをClaude・Codex両方へsymlinkで接続して保証する。正本は plan.md（このHTMLは表示用）。

新書式 行の一生 skill構造 フック plans規約 段階 レビュー項目

要点

ボードは生きている行だけ

チェックボックス廃止。完了＝行を削除して「終わったこと」へ移動。1日100セッションでも完了分は消えるので並ばない。

完了は人間が一言確認

AIが完了判断→「終わったことに入れてよいですか？」と確認→OKで移動。確認が取れなければ⏸のまま残し朝夜会で掃引。

手順は分割・発動は機械

SKILL.mdは索引。開始/終了の手順はreferences 2枚に分割し、フックのスクリプトが対応する1枚だけを確実に注入。

行の一生：登録 → flip → 人間確認 → 移動

新書式：2節・チェックボックスなし

行の型は 時刻 | repo | 種別 | 依頼1行 | 状態。完了した行はここに残らない（削除して移動）ので、ボードは常に「今」だけを映す。

語彙と書かないもの

種別＝計画/実装/レビュー/その他。状態＝🟢動作中/⏸停止・確認待ちの2値のみ。行末の <!-- s:xxxxxxxx --> はフックが自行を特定して機械flipするための短キー（session id先頭8桁・mdレンダリングでは非表示）。完全なsession ID・transcriptパス・SHA羅列・secretは書かない（git/transcriptが正本）。subagent・headless・雑談は登録しない。

skill構造：SKILL.mdは索引、手順は開始/終了で分割

開始時と終了時で読むものが違うので references を2枚に分ける。フックは対応する1枚だけを注入する＝コンテキストが小さく、手順が混ざらない。

skills/session-board/                # 新設（手順の正本）
├── SKILL.md                     # 索引＋共通語彙（行の型・種別/状態・書かないもの）。手順本文は書かない
├── references/
│   ├── session-start.md         # 開始手順: repo特定→種別判定→1行要約→board.sh add
│   └── session-end.md           # 終了手順: 完了判断→人間確認→行削除＋終わったこと追記(git照合)
└── assets/
    └── daily-template.md        # 2節テンプレ

完了確認の文言（session-end.md の核）

依頼（最初＋途中追加分）が終わったとAIが判断したら——「この依頼は完了と判断しました。デイリーの『終わったこと』に入れてよいですか？」（成果の箇条書き案を添えて確認）。OKで行削除＋移動。返答なし/NGなら⏸のまま残す。

フック：正本は hooks/、Claude・Codex両方へ symlink 接続

スクリプト本文はツール側にコピーしない。今朝整理した「本文=hooks/正本／登録=人間ゲート」の方針のまま、目的を持った最小構成だけを戻す。

hooks/session-board/                 # 機構の正本
├── start-inject.sh                  # SessionStart: session-start.md を機械注入
├── stop-guard.sh                    # Stop: (a)自行を⏸へ機械flip (b)報告なしなら1回ブロックし session-end.md 注入
├── board.sh                         # 行upsert・flip・報告検査（flock）。skillからも同じものを使う
└── README.md                    # 登録スニペット（両ツール）・テスト手順

接続（登録は人間ゲート）
├── Claude: ~/.claude/settings.json の SessionStart / Stop → 正本を参照（symlink可）
└── Codex : config.toml notify(turn-ended) → stop-guard.sh 相当   # 開始注入の可否はP3実測
なぜこれで「長いセッションでも確実」なのか

状態flipと報告有無の検査はLLMを介さないスクリプトなので、文脈圧縮や忘却の影響を受けない。判断が要る部分（完了判断・成果要約）だけをセッションにやらせ、やっていなければstopを1回ブロックして手順書を注入する（公式のdecision:block機構・無限ループ防止付き）。フックから別AIを呼ばないのでトークン追加コストなし。subagent/headlessは環境変数ガードで除外。

plans規約：置き場をバケットで統一（planningを全側に）

計画・実装セッションは開始時にrepoを判定して置き場を決める。フォルダ位置＝状態で、md内に状態を書かない。

実作業repo（仕事・focusmap 等）/
└── plans/                # 無ければセッションが作る
    ├── planning/         # 計画セッション: まずここに計画mdを書く
    ├── active/           # 実行に移る時フォルダ移動
    ├── paused/
    └── done/             # 完了で移動

personal-os / my-brain 系
└── my-brain/areas/<領域>/plans/
    └── planning/          # 新設: my-brain側にも追加（areas/AGENTS.md §3・§4.1 改訂）
