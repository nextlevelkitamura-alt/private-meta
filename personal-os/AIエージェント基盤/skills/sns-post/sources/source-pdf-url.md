# source-pdf-url — PDF/URL から求人投稿生成

主に nextlevel__career の ① 具体求人紹介 で使う。
PDF または既存求人URL から職種・場所・給与・期間・条件を抽出 → 投稿文に整形。

## 入力ソース

ユーザーから以下のいずれかを受け取る：
- **PDF**: ローカルパス or 添付ファイル
- **URL**: e-nextlevel 管理画面の求人URL（`https://www.e-nextlevel.jp/master/work/...`）
- **テキスト貼り付け**: 求人内容のテキスト直接

## フロー

### Step 1: ソース読み込み

### PDF の場合
```
Read tool で直接 PDF を読み込む（Claude のPDF読解機能）
ファイルが10ページ以上の場合は pages 引数で範囲指定
```

### URL の場合
- **e-nextlevel 管理画面URL**: 認証が必要なので、ユーザーから「内容コピペで貼って」と依頼するか、Playwright MCP が接続済みなら自動取得
- **公開URL（求人サイト等）**: WebFetch で取得 → 構造化データを抽出

### テキスト貼り付け
そのまま内容を解析。

### Step 2: 必須項目の抽出

以下を抽出する。情報が無い項目は空欄のまま：

| 項目 | 例 |
|---|---|
| 職種 | データ入力＋来客対応 |
| 場所 | 渋谷駅徒歩5分 |
| 時給 | 1,800円 |
| 期間 | 週2〜OK / 単発OK / 長期 |
| 応募条件 | 未経験OK / PC基本操作 |
| 特徴 | 服装自由 / 駅近 / 残業なし |
| 募集人数 | 5名 |

### Step 3: 投稿スタイル選択 → テンプレ流し込み

**nextlevel__career の場合は必ず AskUserQuestion で2スタイルから選ばせる:**

```
どちらのスタイルで投稿しますか？

A. 整え型     — 情報整理重視・公式アカウント感
B. つぶやき型 — 個人投稿感・ぼかし系・問いかけ
```

#### A. 整え型テンプレ
```
📍 {場所}
💰 時給{時給}円
📅 {期間}

「{職種}」のシンプル業務。
{特徴・1行}

気になる方はDMで詳細送ります🙋
```

#### B. つぶやき型テンプレ（推奨・反応取りやすい）
```
{駅}駅徒歩◯分の{業種/場所ぼかし}で
{ざっくり仕事内容}できる人探してます

時給{X}円・週{N}〜・{特徴1つ}
気になる人いたらDMで詳細送ります👌
```

**つぶやき型の生成ルール（厳守）:**
- 場所は「駅名＋徒歩◯分」までで止める（具体ビル名・会社名・現場名は出さない）
- 仕事内容はざっくり1〜2行（業務リスト化しない）
- 条件は1行に圧縮（「時給・週何回〜・特徴1つ」）
- CTAは軽め（「気になる人いたらDM」「ピンときたらDM」「ちょっと話聞きたいだけでもDM」）
- 見出し型（【急募】等）禁止
- 絵文字は最後の1〜2個だけ（過多禁止）
- 「〜いませんか？」「〜したい人いますか？」の問いかけ調を推奨
- 業界・職種感のあるカジュアルな口語（「やつ」「とこ」等は使い過ぎない・1文1個まで）

つぶやき型の例:
```
池袋駅徒歩5分のオフィスで
受付＋簡単な事務できる人探してます

時給1,700円・週3〜・服装自由
気になる人いたらDMで詳細送ります👌
```

### Step 4: 画像の扱い

- 管理画面の求人画像があるなら URL を C列に書き込む
- 無い or 別画像が望ましい場合 → mode-generate-image.md で生成
- 求人系は写真ありが伸びる傾向（accounts ファイル参照）

### Step 5: 投稿日時の決定

accounts/{accountName}.md の `postingTimes` と週間ローテーションから決める。
週3〜4本投稿の場合、月・水・金 の 9時 / 13時 / 18時 など。

### Step 6: スプシ追記

mode-create-post.md の Step 5 に従う。F列=accountName、G列=「① 具体求人紹介」。

## 注意点

- **誇大広告NG**: 「絶対」「必ず」「楽勝」など実態と異なる表現は使わない
- **個人情報マスク**: 特定の担当者名・電話番号を投稿に含めない
- **法令遵守**: 派遣業の場合、職業安定法・労働者派遣法の表示義務を満たす（雇用形態・派遣先業務 等）
- **複数求人をまとめない**: 1投稿1求人が原則

## 既存スクリプト連携

- `scripts/buffer/src/generate-images.ts` — 画像生成（モード3経由）
- `scripts/job-create/` — PDFから求人を作る既存フローがあるが、本スキルの目的は投稿文作成なので、抽出機能のみ参考
- e-nextlevel 管理画面スクレイプ: 認証が必要、必要なら playwright MCP 接続を提案
