# 修正02: ボード視認性の全面改善（人間フィードバック反映）

## 修正項目

1. **「きみの番」レーンを廃止する**
   - board/page.tsx・board-summary-panel.tsx から AskLane の描画とサマリ帯の「きみの番 N」バッジを除去。
   - ask-lane.tsx は削除（未使用部品を残さない）。types.ts の AskItem/asks も build.ts から除去。
   - 質問・確認待ちのセッションは、所属テーマのセッション行の待機表示（琥珀色）だけで示す。
     回答導線（QuestionAnswer）はセッション行タップ展開の中に置く（レーンとして独立させない）。

2. **テーマカードは通常状態＝1行サマリに畳む（デフォルト折りたたみ）**
   - 通常表示はこれだけ: テーマ名・進捗%・稼働N（緑点）・待機N（琥珀点）・完了/総タスク数。
   - タップ/クリックで展開して初めて: やること一覧・ステップ時系列・セッション行を表示。
   - 折りたたみ状態はカード単位の useState（ページ遷移で畳み直しでよい。永続化しない）。
   - PCサイドバーも同一部品なので自動で同じ挙動になる（個別実装禁止・条件7維持）。

3. **完了条件・目的・計画の中身は通常状態のボードに出さない**
   - ボード上の計画表現は「計画チップ（計画名のみ）」まで。チップのタップで plans/[slug] 詳細ページへ飛ぶ。
   - 子02が実装した plans 詳細ページ（md文書タブ）が完了条件・目的・本文の唯一の閲覧場所。
   - テーマカード展開時も計画本文は展開しない（チップ→詳細ページの2段導線）。

4. **視認性の底上げ**
   - 階層は「テーマ（畳み）→ 展開でタスク → タスク内にステップ・セッション」の3段だけにする。
   - 通常状態の1画面あたり情報量を現状の1/3以下に落とす（375pxで全テーマが1画面に収まること）。
   - 未分類枠（StrayBox）も同様にヘッダ1行＋件数バッジで畳み、タップで展開。

## 検証

- tsc --noEmit エラー0・/api/board/summary 200
- 375px目視: 通常状態で全テーマカードが1画面内・詳細はゼロ表示
- PCサイドバー: 同一部品で同じ折りたたみ挙動
