# standup（朝の儀式）

1日を区切る3儀式（Standup 朝・対話3分 ／ Wrap 18:20・対人 ／ Shutdown 23:00・全自動）の朝担当。
前日の完全クローズ→当日の継承を保証し、「今日すること」3〜7業務を人間との短い対話で確定する。
儀式の手順正本はこの SKILL.md のみ（scheduled task 側・launchd 側は本文コピーの無い薄い起動役）。

## 読む面と書く面

- 読む: 前日デイリー（`ゴール/デイリー/<年>/<月>/<日>.md`）・当月の月間計画（`ゴール/デイリー/<年>/<月>/月間計画.md`）。
- 書く: 当日デイリー（今日すること・質問キュー）・前日デイリー（①の締めのみ）・月間計画（⑦の週大枠と✓のみ）。
- それ以外のファイルは書かない。

## ⓪ キャッチアップガード（最初に判定・省略禁止）

起動時刻が **17:00 以降** なら、当日ドラフトは作らない。前日の封印状態＋当日デイリーの現況（今日すること・動いているエージェント・質問キュー）の要約だけ返して終了する。
scheduled task はスリープ復帰時に直近1回をキャッチアップ発火するため、夕方〜夜の遅延発火で「朝の計画立て」が暴発するのを防ぐ二重目のガード（1つ目は scheduled task 側プロンプト文）。

## 手順（平常・朝）

1. **前日封印の確認（最終防衛線）**: 前日デイリーの `## クローズ` 節に `closed HH:MM` 行があるか見る。
   - あれば ② へ。
   - 無ければ（Shutdown が飛んだ日）、先に前日の締めを実施する:
     a. 達成率確定 — 前日「今日すること」の各業務行の最終%と根拠を確定する（根拠は前日「終わったこと」の実在行の引用。推測で盛らない・同日単調・100%は finish 連動）。
     b. 明日へ生成 — 未達業務の持ち越しを前日 `## 明日へ` に追記する（既にある行と重複させない）。
     c. 封印 — `board.py close` で封印する（closed 行・統計の書式正本は board.py／session-board README）。close 未実装期間（子02段階4前）は封印を実行せず「前日未封印のまま進める」と人間へ明示する（closed 行を手書きで偽造しない）。
2. **継承リストの収集**: 前日の `## 明日へ`／未達業務（%と根拠つき）／`## 質問キュー` の未消化分を集める。当日デイリーに `auto:tomorrow-carry` 転記区画（機械転記）があればそれも材料にする。
3. **月間計画の読み込み**: 当月 `月間計画.md` の `## 今月の的`・`## 週の大枠`・`## 予定`（当日分）を読む。
4. **今日することドラフト生成**: 3〜7業務（半日〜1日粒度・「計画を立てるだけ」も1業務可）を業務行の書式（完了条件・親slug・b:key つき。書式は上の正本ポインタ）で起案する。継承業務は前日最終%を引き継ぎ、新規は0%。月間の的・予定から漏れている候補があれば**追加提案**として1〜3件添える。ドラフトはまずチャットに提示する（この時点ではファイルに書かない）。
5. **ズレ調停**: 親リンク切れ（`親:` の slug が月間計画に無い）・ladder矛盾（今日の業務が月間・年間の的に繋がらない）を検出したら、**2択＋推奨つき**で人間に質問する（旧・逆算Alignmentチェックの後継）。自分で勝手に的の文面を直さない。
6. **確定**: 人間の承認・修正を受けて、当日デイリー `## 今日すること` へ業務行を書き込む（board.py の業務行コマンドが実装済みならそれ経由・b:key で冪等upsert）。前日から未消化の質問は当日 `## 質問キュー` へ引き継ぐ。
7. **月曜拡張（Weekly Review 5分版）**: 月曜起動時、月間計画 `## 照合記録` に先週分の Weekly ✓ 行が無ければ、④の前に週の大枠を裁く — 先週大枠の残の処置と今週大枠の確定を人間と5分で決め、月間計画へ反映し `## 照合記録` に✓1行だけ残す。

## 書き込み規律

- 追記が基本。人間の的の文面・手書き行は書き換えない。機械専用行（`進捗:`・業務行の%）は board.py 系コマンド経由でのみ触る。
- `auto:*` 区画へ手で書かない。デイリーの節見出し8つ（正本=テンプレ）を増減・改名しない。
- git commit / push はしない。
- 業務行・的の書式を本文へコピーしない（常に正本ポインタを参照）。
