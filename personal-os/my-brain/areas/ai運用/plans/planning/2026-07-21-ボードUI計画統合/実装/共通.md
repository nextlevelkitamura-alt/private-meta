# 実装担当が全子で守ること

- 憲法（program.md「正本境界」4条）を最優先する。違反しそうになったら実装を止めて報告する。
- ファイル担当マップ（program.md）の自分の範囲外を触らない。必要になったら停止して該当子へ差し戻す。
- 導出値（%・今ここ・稼働N体）をDBに保存しない。全てSQL導出。
- migration適用・push・hook登録変更は準備止まりで停止（人間ゲート）。ファイルは作ってよいが適用しない。
- Turso本番への書き込みテスト禁止。session-boardのfake DB（tests/_fakedb.py）または SESSION_BOARD_NO_TURSO を使う。
- secret・token・認証値を表示・記録・commitしない。
- UIは白背景ライト単色・375pxモバイルファースト・固定幅flex禁止。実行中の明滅は2.4秒周期（人間調整済み・変更しない）。
- 完了時は result packet（status / base_commit / result_commit / changed_paths / tests / 設計判断 / blockers / remaining_risks / out_of_scope_findings）で返す。

## 評価者（レビュアー）の観点   ※ 評価mdは `../評価/NN-〈子名〉-評価RR.md`

- 完了条件を1項目ずつ実物（コード・テスト実行・実データ）で照合する。文書だけでPASSにしない。
- 憲法違反（DB→md書き戻し・第2状態台帳・導出値の保存・pathリンク）を最優先で探す。
- 担当マップ越境（他子のファイルを触っていないか）をdiffで確認する。
- 沈黙故障（リンク切れ・steps未登録・捕捉失敗が黙って空になる）を許さない。可視化されているか。
- hook非ブロッキング（内部失敗で本体を止めない）をテストで確認する。
- 評価カデンス: 子01〜03は都度、子04はprogram最終評価と同時にまとめて。
