# 実装評価修正ループ（計画→実装→評価→修正のMD駆動化）

優先: ◎

## 目的

実装委任（/codex-impl・Codex exec直接駆動）を、既存の規模別レビュー制度（運用契約§2・決定ログ#9）に接続する。指示の口頭伝達は劣化するため、サイクルの各段を**MD文書で駆動**する: 計画（plan.md）を読ませて実装し、評価（評価NN.md）で採点し、FAILなら修正指示書（修正NN.md）を作って差し戻す。サクッと（構造3条件全YES）はこのループを通さず単発実行のまま。

## 全体像

```
規模判定（サクッと3条件）
├─ サクッと → 直実行 → diff要約を人間へ（計画・評価なし）
└─ ライト以上
   plan.md（完了条件=レビュー項目・着手前に必須）
   → Codex実装（依頼文冒頭で plan.md を読ませる指定を必須化）
   → impl-reviewer（claude系sub・項目×diff採点）→ 評価NN.md
   ├─ 全PASS → 人間確認 → done
   └─ FAIL → 修正NN.md（対象・今の状態・期待状態・方法を明記）
             → codex exec resume <thread_id>「修正NN.mdを読んで修正」
             → 再評価（差し戻し上限: ライト1・フル2 → 超えたら人間へ）
```

命名: 単発計画は plan.md と同フォルダに `評価01.md`/`修正01.md`/`評価02.md`…。programの子は `plans/NN-子名-評価01.md` のように子と同じ接頭NNで並べる。ラウンド番号=差し戻し回数。
