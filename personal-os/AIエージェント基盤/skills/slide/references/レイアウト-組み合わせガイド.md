# レイアウト組み合わせガイド

8種類のレイアウトを、スライドマップのどこに配置するかの推奨パターン集。

## 基本原則

1. **同じレイアウトを連続させない**（読み手が飽きる）
2. **章扉と本論でレイアウトを変える**（メリハリ）
3. **キーメッセージはミニマル系で印象づける**
4. **データは「データ表-グラフ大」を独立スライドに**

## 構成テンプレ別の推奨レイアウト

### PASTOR（課題ドリブン）
```
[1] Problem                  → 全画面写真（問題の象徴）
[2-3] Problem 詳細           → 見出し本文中心 / 3カラム比較
[4] Amplify                  → ミニマル文字のみ（衝撃の一言）
[5-7] Solution               → 右画像-左文字 / 図解中心
[8] Testimonial（データ）     → データ表-グラフ大
[9] Offer                    → 見出し本文中心
[10] Response                → ミニマル文字のみ（CTA）
```

### 営業提案
```
[1] 表紙                     → 全画面写真 or ミニマル
[2] 本日のゴール             → 見出し本文中心
[3-4] 御社の現状理解         → 図解中心 / 3カラム比較
[5-6] 課題仮説              → ミニマル文字のみ（仮説1文）+ 見出し本文
[7-9] 解決策                → 右画像-左文字 ×3
[10] 導入事例               → データ表-グラフ大
[11] ROI                    → データ表-グラフ大
[12] スケジュール            → タイムライン
[13] ネクストアクション       → 見出し本文中心
```

### 講演登壇
```
[1] Hook                    → 全画面写真 or ミニマル文字のみ
[2] 共通体験               → 見出し本文中心 + 写真
[3-4] 転換点（実は…）       → ミニマル文字のみ + 全画面写真
[5] 核心メッセージ          → ミニマル文字のみ（STAR Moment配置）
[6-9] How（具体的方法）     → 右画像-左文字 / 図解中心
[10-11] ビジョン            → 全画面写真
[12] One Action             → ミニマル文字のみ
```

### 自己紹介
```
[1] 表紙（名前+キャッチ）    → 全画面写真（自分の写真）
[2] プロフィール             → 見出し本文中心
[3-4] 今やっていること       → 3カラム比較
[5-6] 実績・数字            → データ表-グラフ大
[7] Why                     → 全画面写真 + ミニマル
[8] 一緒に作りたい未来       → ミニマル文字のみ
[9] 連絡先                  → 見出し本文中心
```

### 圧縮（5-7枚）
```
[1] Answer                  → ミニマル文字のみ
[2-3] Why（理由3点）         → 3カラム比較
[4-5] Data                  → データ表-グラフ大 ×2
[6] Risk                    → 見出し本文中心
[7] Decision                → ミニマル文字のみ
```

### 社内共有
```
[1] サマリー                 → 見出し本文中心（このスライドだけで完結）
[2] 背景                     → 見出し本文中心
[3-4] 結果・進捗             → データ表-グラフ大 / 3カラム比較
[5-6] 気づき                 → 見出し本文中心
[7] 今後の予定               → タイムライン
```

## 例外: 全部ミニマル統一

講演ベースで「世界観で勝負する」場合は、**全スライドをミニマル文字のみ + 全画面写真の2種だけ**で構成する選択肢もあり。
スティーブ・ジョブズ型・TED の世界観プレゼン。

## NGパターン

❌ **3カラム比較を3枚連続**: 視覚的に単調
❌ **データ表を1スライドに2つ**: 認知負荷オーバー
❌ **見出し本文中心を5枚連続**: 配布資料っぽくなる
❌ **章扉に見出し本文中心**: 章転換のメリハリが出ない
❌ **CTA（行動喚起）に図解中心**: 行動が埋もれる

## 推奨パターン

✓ **章扉**: 全画面写真 or ミニマル文字のみ
✓ **キーメッセージ**: ミニマル文字のみ
✓ **本論の説明**: 右画像-左文字 / 図解中心
✓ **データ**: データ表-グラフ大（独立して使う）
✓ **比較**: 3カラム比較
✓ **時系列**: タイムライン
✓ **CTA**: ミニマル文字のみ
