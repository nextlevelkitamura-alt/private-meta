# 朝夜ルーティン

毎日、目標(ゴール)・ダッシュボード・デイリーを突き合わせ、逆算Alignmentと状態更新を回す手順。
当面は手動。skill化の要否は program A 子04(判断系スキル統合)で束ねて判断する（今は増やさない）。

## 朝（5分）
1. `../ダッシュボード.md` を開く（計画active / AIジョブ / 次の一手を俯瞰）。
2. `年間計画/<年>.md` と `3年計画.md` の的を確認。
3. 今日のデイリーを作る: `templates/デイリー.md` をコピー → `デイリー/<年>/<月>/<年-月-日>.md`。
4. 「今日やる」を**逆算で**選ぶ（年間/3年の的に繋がるものを優先）。

## 夜（5分）
1. デイリーの「できた」に実績を追記。
2. 「逆算チェック」を埋める（下記）。
3. ダッシュボードを更新（計画の状態が動いたら表の行・次の一手を直す。当面は手更新、将来 dashboard.sh）。

## 逆算Alignment（夜のチェック）
- [ ] 今日やったことは年間/3年目標に繋がっていたか。逸れていたら明日へ補正。
- [ ] 対応の無い放置目標はないか（的はあるのに今週デイリーに一度も出ていない目標）。
- [ ] `running` 滞留はないか（基盤 `ai-jobs/running/` に長居している run-card）。
- [ ] 古い `active` はないか（`plans/active/` で次の一手が何日も動いていない計画）。
- 検出したら: ダッシュボードの「次の一手」に書く、または run-card / 計画へ反映する。

## 関連
- 的と履歴: このフォルダ `ゴール/`（3年/年間/デイリー）。
- 状態の1枚: `../ダッシュボード.md`。
- 自動化(将来): `dashboard.sh` が plans/ と ai-jobs/ から状態を再生成（子02-B）。逆算Alignmentの「放置検出」もここに寄せられる。
