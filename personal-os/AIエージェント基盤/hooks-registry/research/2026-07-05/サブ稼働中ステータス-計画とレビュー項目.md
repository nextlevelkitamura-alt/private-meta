# 🔵サブ稼働中ステータス — 実装計画とレビュー項目（2026-07-05）

## 目的

バックグラウンドのサブエージェントが実行中の間、mainセッションのボード行を
「⏸停止・確認待ち」にせず「**🔵サブ稼働中**」で区別する。
「裏でサブが動いているのに停止扱い」という実態との乖離を解消する。

## 現状

- 状態は2値（`🟢動作中` / `⏸停止・確認待ち`）。
- Stop（`claude/session-end.py`）が🟢→⏸へ機械flip。サブ実行中でも⏸になる＝実態とズレる。

## 方針（実装計画）

状態を**3値化**。機械自動判定（フックがサブ実行中を検知）は難しいため、**エージェント明示方式**を採る。

1. **board.py**:
   - `SUB = "🔵サブ稼働中"` 定数を追加。
   - `LINE_RE` の state を `🟢動作中|⏸停止・確認待ち|🔵サブ稼働中` に拡張。
   - `flip --state sub` で🔵に遷移。`check` の出力に `sub` を追加（`missing|run|wait|sub`）。
2. **claude/session-end.py**（Stop）: 状態が `sub` なら flip しない（⏸にしない＝サブ稼働中を維持）。`run` のときだけ⏸。
3. **claude/prompt-register.py**（UserPromptSubmit）: `wait` は🟢復帰。`sub` は触らない（サブ稼働中を維持）。`missing` は新規登録。
4. **session-start.md**（手順）: 「バックグラウンドのサブエージェントを起動したら `board.py flip --state sub`、完了通知を受けたら `--state run`（継続）に戻す」を追記。
5. **README.md / hooks/AGENTS.md**: 3状態を記載。

## レビュー項目（Opus検証用・「こうなっていれば正しい」で書く）

1. **board.py**: `SUB`定数・`LINE_RE`のstate拡張・`fmt`/`flip`/`check` が sub 対応。既存2状態（run/wait）も従来どおり動く。
2. **flip sub**: `flip --key K --state sub` で行が `🔵サブ稼働中` になり、`check` が `sub` を返す。
3. **session-end.py**: 状態 `sub` のStopで⏸にflipしない（sub維持）。`run` は⏸。`missing`/`wait`/`sub` で誤動作しない。
4. **prompt-register.py**: `sub` の行はプロンプト送信で🟢復帰しない（sub維持）。`wait` は🟢復帰。`missing` は新規登録。
