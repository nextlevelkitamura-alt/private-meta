repo-create

新しいrepoの最小構成を作る、または既存repoを安全に評価・改善する入口です。repo横断移植では、書き込む前にcanonical identityと計画箱を解決し、値を出さない監査契約を使います。

いつ使うか（発火する発話例）

  • 「新しいrepoを作って」「既存repoをAIが作業しやすい状態へ整えて」
  • 「AGENTS.md / CLAUDE.mdを見直して」「remoteやupstreamを確認して」
  • 「別repoへAI運用標準を移せるか、まず安全に監査して」

構造

各workflowの中身

create-repo.md — 新規repo

目的・公開範囲・置き場を決め、AGENTS、CLAUDE symlink、README、gitignoreを中心に最小構成だけ作ります。

目的と種別→Git / public / push承認→status確認

improve-repo.md — 既存repo

Git状態と入口文書をread-onlyで確認し、「今すぐ直す／整える／任意」に分類して必要最小限だけ編集します。

read-only監査→最小差分→整合確認

横断移植の位置づけ: 現時点では契約とfixtureまでです。audit-repo などの実行scriptは後続packageで実装するため、契約だけを見て「監査コマンドが利用可能」とは扱いません。

referencesの中身

agents-md-criteria.md — repo入口の評価基準

  • 共通構成: 目的、禁止、フォルダ、開始・更新・確認方法
  • repo種別: 業務運用repoとcoding repoで必要な説明を分ける
  • AI runtime構成: Codex/Claudeを使うrepoの境界。repo-local skillは.agents/skills/<name>/SKILL.mdを正本にし.claude/skillsはそこへのsymlink（1 skill 1窓）。Codex固有hooks/rules/custom agents/configは.codex/に限定しskill本体を置かない（詳細はglobal-skill-registry/AGENTS.md）
  • 評価: 初見の分かりやすさと危険操作の停止可否

repo-safety.md — 副作用と人間承認

  • Git: 未コミット差分を保護し、remote・main・force操作を承認制にする
  • Secret: 値を表示・記録・commitしない
  • 配置: repo本体をPrivate直下へ増やさず、物理移動はrepo-relocationへ渡す

migration-contract.md — 横断移植の決定的監査契約 K01

  • Identity: registry、Git common dir、worktree/container、未mountを区別してdedupe
  • 計画箱: repo AGENTSを正本にし、未宣言・曖昧ならexit 3でfail-closed
  • 出力: 同じJSON envelope、固定sort、exit 0/2/3/4、secret値出力0
  • Fixture: 仕事・focusmap・symlink・計画箱・未mountなど15件の合成ケース

絶対ルール・安全方針

既存ファイルを勝手に上書きしません。 削除・移動・改名・commit・push・hook/launchd登録は契約だけでは許可されません。CLAUDE.md は同階層の AGENTS.md への相対symlinkを基本とし、固有内容がある時は先に損失なく統合します。secretは存在と件数だけを扱い、値は出力しません。
