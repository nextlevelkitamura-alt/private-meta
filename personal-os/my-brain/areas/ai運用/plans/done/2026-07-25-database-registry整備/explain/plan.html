AIエージェント基盤 / 表示専用の構造説明

database-registry整備

接続情報を集める場所ではなく、各サービスの正本へ安全に辿り着くための案内を作る。値・接続URL・現在状態は保存しない。

どこを触るか

基盤内に案内専用のフォルダを1つ追加し、既存repoの設定や認証情報には触れない。

AIエージェント基盤/

既存の基盤入口。フォルダ一覧だけ更新。

→

database-registry/

共通の安全規約と接続案内。

providers/

Supabase / Turso / Cloudflare / Cloud Run。

local/

gitignoreされたMac専用の補助設定。

こう変わるか

散在する設定を移動せず、確認するときの入口だけを一箇所にそろえる。

変更前repoごとに環境変数・CLI・コンソールの入口を探す。
→
変更後database-registryでproviderを選び、正本と安全な確認順を辿る。
→
正本対象repoと公式コンソールでだけ実状態を確認・変更する。

中身の意味

各provider文書は「どこが正本か」「何なら安全に確かめられるか」「値はどこへ置くか」だけを短く示す。

Supabase / Turso

Dashboardと対象repoを正本にし、CLIはhelp・バージョン確認から入る。

Cloudflare

WorkersとR2を同じWrangler案内にまとめる。R2のS3互換は別種のキーになり得る。

Cloud Run

Google Cloud Consoleと対象repoのbuild/deploy定義を正本にする。

守る境界

残すもの: 値なしの案内、正本へのリンク、非破壊の確認手順。
残さないもの: secret、token、環境変数値、接続URL、実プロジェクト名・数、CLIの認証出力。
