# 06 安全な操作窓口

## 目的

read-only AI運用ハブの観察後、人間が本当に必要と判断した定型操作だけをFocusmapから依頼できるようにする。任意shellや無承認の停止 / 再開は許可しない。

## 現状

- Focusmap agentにはcommand loopがあるが、pause / resume / restart / upload_logs / cancelの一部はackだけで実体未実装。
- WebからローカルMacへ直接触れる経路はplatform境界上避けるべき。
- hook / launchdの追加・削除・停止・再開は人間ゲート。

## 方針

### 操作モデル

```text
Focusmap UI
  → 固定 unit_id + 固定 action + idempotency key のintent作成
  → 人間確認（対象・効果・rollbackを表示）
  → local consumerがallowlist / 現在状態 / gateを再検証
  → 定義済み作用器だけを実行
  → 結果と監査eventを返す
```

- default OFFのfeature flag。
- 初期allowlistは `dry-run` / `再診断` / `ログ要約要求` など非破壊操作から始める。
- pause / restart / enable / disableはunitごとに人間ゲートを要求する。
- 未知unit ID、未知action、任意command文字列、path指定は拒否する。
- UIとconsumerの両側でallowlistを検査する。UI非表示だけを安全策にしない。
- actionごとにprecondition、timeout、idempotency、success evidence、rollbackをmanifestへ持つ。
- token / env / raw logはintentや監査eventへ含めない。

## 実行ゲート

- 子03のread-only UIが最低1週間安定。
- 子04 / 08のmanifest / verify、子05のFocusmap安全化、子09のGlobal整合保証が全PASS。
- 子07のsnapshot認可・保持・情報公開allowlistが全PASS。
- 人間が初期allowlistを1件ずつ承認。
- DB schema / token / local consumer loadは別々に承認。

## 完了条件（レビュー項目）

- [ ] feature flag OFFではintent作成・consumer実行が0件である。
- [ ] 未知unit ID、未知action、任意command / pathをAPIとconsumerの双方が拒否する。
- [ ] 同じidempotency keyの再送で作用が1回だけ実行される。
- [ ] 破壊性のあるactionは人間確認なしで実行されず、対象・効果・rollbackが確認画面に出る。
- [ ] success evidenceが無い時は成功表示せず、failed / unknownとして残る。
- [ ] intent / event / UI / logにsecret値とraw credentialが0件である。
- [ ] 操作面をOFFにするだけでread-only UIと既存runtimeへ戻れる。
- [ ] 敵対レビューで任意shell、ID偽装、再送、stale状態、consumer停止の各ケースを防げる。
