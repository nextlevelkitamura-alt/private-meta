# PM階層との対応（参照）

```text
portfolio  >  program  >  project   >  task
   ≒ area   >  program  >  単発計画  >  ai-jobs の run
```

- 命名と判定基準の根拠メモ。読む用の確定材料（考え・未確定は plan.md の `方針` / `identity.md` へ）。
- 「program」はPMで「束ねて初めて価値が出る、関連 project の集合」。この階層の正式名。
- 判定基準: 独立に卒業する子計画を2本以上生む → program 化。派生が単発作業で足りる → 単発計画のまま。
