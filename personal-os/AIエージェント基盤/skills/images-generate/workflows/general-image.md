# 一般画像の生成

一般的な画像、サムネ、バナー、アイキャッチ、SNS図解などを生成する。

## フェーズ0: 5項目チェック

生成前に次の5項目が揃っているか確認する。

| # | 項目 | 例 |
|---|---|---|
| 1 | アスペクト比 | `1:1`、`16:9`、`7:5`、`4:5`、`9:16` |
| 2 | 主役・ビジュアルの核 | 人物、商品、風景、図解の中心要素 |
| 3 | テイスト | リアル写真風、イラスト、アニメ、ミニマル |
| 4 | 用途・媒体 | note、SNS、LP、スライド |
| 5 | テキスト要素 | なし、短いタイトル、バッジ |

不足項目が複数あっても、選択肢や例を添えて1回のメッセージにまとめて聞く。すでに分かる項目は聞き直さない。

## フェーズ1: プロンプト確定

5項目を英語のプロンプトに組み立て、次の形式で確認する。

```text
このプロンプトで生成しますか？

[画像名]
> [完成した英語プロンプト]

アスペクト比: [比率]
テイスト: [テイスト]
用途: [用途]

[このまま生成] / [修正したい]
```

修正依頼では変更箇所だけを聞き直し、全項目を再確認しない。

## フェーズ2: 生成・編集

確定したプロンプトごとに1枚ずつ生成する。

```bash
codex exec --json --skip-git-repo-check -C <作業dir> "image_gen で <英語プロンプト> を生成して保存パスを報告"
```

出力1行目の `thread.started` に含まれる `thread_id` を控える。修正時は新規生成せず、同じスレッドを再開する。

```bash
codex exec resume <thread_id> "<修正指示>"
```

複数枚でも1プロンプト1生成とし、各画像の役割・画面内テキスト・図解要素を個別に書く。

生成後、プロジェクトで使う画像は `~/.codex/generated_images/...` から該当ワークスペースへコピーし、最終保存パスを報告する。

## よくある失敗パターン

| 症状 | 対処 |
|---|---|
| 画像内テキストが崩れる | 入れたい文字を短い英語で指定する |
| バスト・体型が誇張される | `voluptuous` や `large` を避け、`naturally attractive`、`proportionate` を使う |
| メモ書きが背景に出る | `No handwriting, no scattered papers` と明示する |
