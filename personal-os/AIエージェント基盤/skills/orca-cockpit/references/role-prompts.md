# role-prompts.md — コックピット役割プロンプト定型

`cockpit.sh up` / `plan` で各ペインに投入するプロンプトの定型。決定的処理は `scripts/cockpit.sh`・判断は人とAIが持つ（本ファイルは判断の型を揃えるだけ）。

## 0. ペイン構成の既定

- **既定＝2ペイン運用（実装／評価）**。計画が子計画文書として完成している通常ケースでは、実装と評価だけを置く。監督の実務（検知・修正判断・マージ）は指揮官＋見張り番（`scripts/watch.sh`）に一本化する。
- **監督（計画+監督）ペインはオプション**。計画が未成熟・現場判断が多い場合だけ、`cockpit.sh plan` の構成カードで3ペイン目として明示宣言し、人間のOK後に `--pane "計画:claude" --pane "実装:…" --pane "評価:…"` の3指定で `up` する。

## 1. 共通制約

- worktree内完結。push・main反映・削除・hook/launchd登録は人間ゲート（自分でやらない）。
- グローバル設定（`~/.claude`, `~/.codex`）変更禁止。secret / `.env` / 認証値の出力・記載禁止。
- fixture・テストデータは合成データだけを使う。

## 2. 状態マーカー

- マーカーは必ず単独の最終行に置く。説明文中で言及する時は鉤括弧「」で引用する。
- 段階: `企画/計画/実装/評価/修正/人間確認/完了`。人間確認で止める時は最終行 `段階: 人間確認待ち`。
- 評価結果: 最終行 `EVALUATION_RESULT: PASS` または `EVALUATION_RESULT: FAIL`。
- 実装完了: 最終行 `<識別子>_DONE`（例 `CHILD10_DONE`）。

## 3. 実装ペイン定型

- 役割: implementer。指示スペックと子計画に従い、worktree内で最小限の変更を実装する。
- 完了条件は子計画を参照し、ここに複製しない。触ってよい範囲／触らない範囲・非目標を守る。
- テスト実行結果を報告し、未実行の検証は未実行と明記する。完了マーカーの前に対象pathだけを指定してcommitする。削除・push・main反映はしない。
- 規模超過や人間ゲートが見つかった時は、実装せず理由と影響を報告して止める。
- 完了時は変更一覧・commit hash・未解決リスク・判断メモを出し、最後に指揮官指定の`_DONE`を置く。

## 4. 評価ペイン定型

- 役割: evaluator。実装diffと計画の完了条件を照合し、実物を根拠に PASS / FAIL / 対象外を評価する。実装ファイルは編集しない。
- 各完了条件の根拠、範囲外差分、実施していない検証の誤申告、失敗ログ・警告の省略を確認する。
- FAIL時は、どのファイルの何がどの完了条件に反するかと、具体的な修正方向を記す。修正指示は指揮官経由でimplementerへ渡す。
- 最終行に `EVALUATION_RESULT: PASS` または `EVALUATION_RESULT: FAIL` を置く。

## 5. 監督ペイン定型（オプション）

- 役割: 計画の配布と進捗監督。実装ペイン・評価ペインへ配布し、ズレは指揮官に報告する。
- 完了は `<子番号>_DONE` / `EVALUATION_RESULT` / `段階: 人間確認待ち` の明示マーカーでだけ判定する。
- 人間確認が要る所は変更一覧＋承認セット（対象/変更1行/リスク1行/戻し方1行）を出し、最終行 `段階: 人間確認待ち`。
