# 儀式の自動実行（Standup / Wrap / Shutdown）

## 目的

Standup（朝・対話3分・自動ドラフトがチャットに現れる）／Wrap（18:20・対人・仕事締め＋AI質問消化）／Shutdown（23:00・全自動・デイリー封印）を定時運転させる。前日の封印→翌朝の継承を機械の契約にする。

## 現状（調査3・2026-07-09）

- Desktop scheduled tasks（Claude Code Desktop・現環境2.1.201で利用可）が「定時にmacOS通知＋サイドバーに新セッション出現→そのまま承認対話」を公式提供。ローカルファイル・MCP可。制約: アプリ起動中＋Mac稼働時のみ（スリープはwake時に直近1回キャッチアップ）。
- Cloud routines はローカルファイル不可（不採用）。launchd＋headless `claude -p` は実績十分だがチャットに現れない→無人のShutdown向き。
- 資産: carry-tomorrow.sh（前日「明日へ」転記・実装済み・眠っている）・ensure-daily.sh・session-record-prune（StartCalendarInterval型loopの最新雛形）・marker I/O・morning-routineスキル（朝会=Standup部分と指揮官編成の2責務が同居）。
- 朝ルーチン自動実行と秘書構想（07-03・滞留中）は方式差し替えでStandupへ吸収。

## 方針

1. Standup: Desktop scheduled task（平日朝）＋Global Skill `/standup`。読む=前日デイリー（明日へ・未達%・質問キュー）＋月間計画（的・週大枠・当日予定）。出す=今日すること3〜7業務ドラフト＋追加提案＋ズレ調停の質問。冒頭で前日未封印を検知したら先に締める（最終防衛線）。月曜はWeekly未実施なら5分拡張版。
2. Wrap: Desktop scheduled task 18:20＋Googleカレンダー定期イベント（スマホ保険・カレンダーMCPで作成）。`/wrap` スキル。仕事系業務の締め＋質問キュー消化＋今夜の1行確認。
3. Shutdown: launchd 23:00（`loops-registry/loops/daily-shutdown/` 新設・session-record-prune の型を踏襲・plist有効化は人間ゲート）。Python=枠（達成率確定・closedマーク・週大枠書き戻し・marker冪等）／LLM=stdout契約のみ（⏸最終判定・明日へ作文。ファイルは書かない）。LLM失敗時はPythonだけで封印まで完了し「LLM未処理」フラグ。commitはしない。`/shutdown` は同じscriptを対話から叩く薄い窓。
4. morning-routineスキルは Standup/Wrap/Shutdown へ分配し、指揮官編成専用に縮退 or 廃止（skill-creator-custom→skill-delete の正規ゲート経由）。儀式の手順正本は基盤 `skills/standup|wrap|shutdown/` のみに置き、`~/.claude/scheduled-tasks/*/SKILL.md` は薄いポインタ。
5. キャッチアップ暴発ガードをプロンプトに書く（例: 17時以降のStandupは要約のみ）。

## 完了条件（レビュー項目）

- [ ] 平日朝、定時＋数分以内に通知が出てサイドバーにStandupセッションが現れ、開くと今日すること3〜7個のドラフトが最初の応答として読め、承認・修正の対話ができる
- [ ] 前日デイリーにclosedマークが無い状態でStandupが走ると、当日ドラフト提示の前に前日の達成率確定〜封印が完了する（Shutdownが飛んだ日でもデータ欠落ゼロ）
- [ ] 23:00のShutdownで `claude -p` が失敗しても、Python枠だけで達成率確定・週大枠書き戻し・closedマークまで完了し「LLM未処理」フラグが残る。同一入力の2回連続実行は差分ゼロ。デイリーへのcommit/pushは行われない
- [ ] 18:20にDesktop通知とカレンダー通知（スマホ）の2経路で届き、人間が開かなくても質問キューは消えず翌朝Standupドラフトに載る
- [ ] 儀式の手順正本が基盤 skills/ のみにあり、scheduled-tasks側・launchd入口scriptは本文コピーの無い薄い起動役。morning-routine側に旧発火語・重複記述が残っていない
