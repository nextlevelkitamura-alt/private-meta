# 実演例3: フル（合成例）

## 入力

- やりたいこと: 「orca-cockpitの権限モデル（役割別 `.claude/settings.json` 配布）を複数repo横断で見直したい」
- 対象repo見当: 不明（横断・複数repoにまたがる）

## 4判断

1. **規模**: 権限モデルの見直し＋各repoへの適用が独立に卒業する子計画2本以上（設計本体＋repo別適用ロールアウト）に割れる見込み → **フル**。
2. **経路**: Private起点でrepo-registryから中央正本repoを選び、対象AGENTSが横断・Global Skill計画として宣言する `~/Private/personal-os/my-brain/areas/ai運用/plans/planning/` を候補にする。既存planが一意なら合流し、複数候補・正本不明ならexit 3で停止する。repo別ロールアウトは中央計画の子として分割し、各repo書込み前に別handoffする。
3. **起動形**: 計画がまだ子計画文書として固まっていない（企画〜計画段階から始める）→ **3ペイン例外**（計画+監督ペインを追加、構成カードで明示宣言）。
4. **モデル**: 問い「このレーンが遅いと誰が待つか」→ 各repoへの適用ロールアウト（他レーン）がこの設計待ちでブロックされる直列ボトルネック → 実装をopus4.8 fastへ格上げ（構成カードで宣言）。レビューはxhighのまま。
