# database-registry 接続案内

## このレジストリがすること

この場所は接続値を集約しない。各サービスの正本へ安全に到達するための案内だけを持つ。

| provider | 使う案内 | 正本 |
|---|---|---|
| Supabase | `providers/supabase.md` | Supabase Dashboardと対象repoの設定 |
| Turso | `providers/turso.md` | Turso Dashboardと対象repoの設定 |
| Cloudflare | `providers/cloudflare.md` | Cloudflare Dashboard、Wrangler設定、対象repo |
| Cloud Run | `providers/cloud-run.md` | Google Cloud Consoleと対象repoの設定 |

## 安全な確認順

1. 対象repoの最寄り`AGENTS.md`を読んで、何を確認してよいかを決める。
2. ファイル名・設定キーの種別・CLIのバージョンだけを確認する。値、接続URL、認証結果の出力は記録しない。
3. 現在の資源や権限を確かめる必要がある時は、公式コンソールを使う。CLIを使う場合も出力をチャット・Markdown・commitへ貼らない。
4. 変更は対象repoの承認済み手順からだけ行い、直後にサービス側でreadbackする。

## 資源を作成・変更する前の確認

次の5点が対象repoの実行契約で明示されるまで、create、migration、deploy、delete、token生成を実行しない。

1. 対象providerと対象project/database/bucket/service
2. 環境（local、staging、本番）と実行権限
3. 変更内容と影響範囲
4. サービス側のreadback方法
5. rollback方法または、不可逆であることの承認

## ローカル設定の置き場

- repo固有の設定は、そのrepoが既にgitignoreしているローカル設定領域を正本とする。
- この基盤横断で補助設定が必要な時だけ、`local/`に置く。ここでは追跡しない。
- sample、説明、テスト結果にも値・接続URL・実プロジェクト情報を書かない。

## 確認できた対象範囲

値を読まずに、Focusmapのローカル設定ファイル名と設定キーの種別だけを確認した。Supabase、Turso、Cloudflare R2、Cloud Runに関係する設定の存在を確認している。Workersの利用有無や各providerの実プロジェクト状態は、このレジストリに記録しない。
