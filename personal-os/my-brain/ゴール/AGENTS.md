# ゴール（縦のladder）

3年→年間→月間→デイリーの「的」と「履歴」を置く縦のladder。
横の領域別は `../areas/`。全体の「今」（動いているエージェント／終わったこと）は board DB（Turso）が正本で、focusmap がDBから描画する（2026-07-21 正本反転・案b＝デイリーmd 2節は廃止）。縦と横を混ぜない。
的は上から下へ落とす（3年→年間→月間→今日・下位はリンクで参照）。実績は下から上へ、機械の専用行だけを通って上がる。

## 構成
- `3年計画.md`: 3年後にありたい状態（的）。
- `デイリー/<年>/年間計画.md`: その年の的。3年計画から落とす（旧 `年間計画/<年>.md` は 2026-07-09 に入れ子へ移設・旧フォルダ廃止）。
- `デイリー/<年>/<月>/月間計画.md`: 今月の面。節は「今月の的／週の大枠／予定／照合記録」の4つ。
- `デイリー/<年>/<月>/<年-月-日>.md`: 1日1枚・追記の履歴（6節）。
- `templates/デイリー.md`: デイリー雛形の正本（6節。見出し文言・順序はここが正。旧「動いているエージェント」「終わったこと」の機械2節は 2026-07-21 正本反転で廃止＝board DBへ一本化）。
- `templates/月間計画.md`: 月間計画雛形の正本（4節）。
- `references/`: ゴール設定インタビュー等の参照テンプレ。

毎日の運用手順の正本は Global Skill `morning-routine`（Standup/Wrap/Shutdown の3儀式への分配は「デイリー運用刷新」program 子03 が担当）。

## 的の行フォーマットとslug規約（ここが正本）

3年・年間・月間・今日の全レイヤ共通。他文書・機構READMEはここを参照する（本文コピー禁止）。

```
- [slug] 内容（条件: どうなれば達成か1行） ｜親: 上位#slug
  進捗: ▓▓▓▓░░░░░░ 40% ｜根拠: … ｜更新: MM-DD
```

- slug は短い1語（日本語可）。一度付けたら改名しない。全レイヤの照合キー。
- `親:` は `年間#slug`・`月間#slug` の形で上位レイヤの的を指す。
- `進捗:` 行は機械専用（人間の的の文面を機械が書き換えることは絶対にない）。進捗%はAI推定で、根拠1行を必ず添える。

## ゴールが未設定/薄いとき
3年/年間が「未確定」だらけだと逆算も月間もデイリーも空回りする。`references/ゴール設定インタビュー.md` の問いに沿って対話で引き出し、固まった分だけ記入する（全部一度に埋めなくてよい）。

## ルール
1. デイリーは1日1枚・6節（雛形は `templates/デイリー.md`）。「動いているエージェント」「終わったこと」の機械2節は 2026-07-21 の正本反転（program「当日ボードSQL化」子03・案b）で**廃止**し、正本を board DB（Turso: sessions/session_logs/session_subagents）へ一本化した。board.py はデイリーmdを読み書きしない（生成もしない）。この2節の閲覧は focusmap（DBから描画）か `board.py show` で行う。共通エンジンの正本は `../../AIエージェント基盤/hooks-registry/shared/session-board/`。
2. デイリーには必ず「逆算チェック」を書く（今日やることが月間/年間/3年の的に繋がるか）。
3. 進捗・実行状態は的の文面に書かない。機械が書くのは `進捗:` 行などの専用行のみ。
4. 目標の中身（3年/年間/月間の的）は人間が記入する。AIは雛形とドラフト（「（ドラフト・人間確認待ち）」明記）まで。
5. 的の文面・構成を変えたら、そのファイル末尾の `## 変更履歴` に1行残す（ズレ調停の記録先）。
6. secret / token / 認証値は書かない。
