## 方針（4段導入）

- 段階1: 今日すること節＋業務行CRUD（biz-add/biz-update/biz-done・%バー描画・途中追加の時刻マーク・テンプレ節保証）。純加算・既存97本無傷。
- 段階2: 紐づけ＋ぶら下がり表示。まず案C（業務名=目標文言の規約強化・フォーマット変更ゼロ・hook注入文に業務一覧）で開始し、不足が実証されたら案B（業務行側メンバーリスト `<!-- b:key s:a,s:b -->`）へ昇格。表示はgoals-summary同型の機械占有ブロック（業務行直下にセッションのミニ行・`<!-- s: -->`無しの非マッチ形式）。test-shims.sh の注入文assert群（8〜12本）はここで意図的更新。
- 段階3: 定型レーン。種別5種は増やさず、定型は業務の属性（`## 定型レーン` 節に業務行を置く）。セッション行は動いているエージェント節に留める。
- 段階4: 封印。`board.py close` サブコマンド（クローズ節＋closed HH:MM＋統計1行）＋封印後の書き込みは翌日ファイルへ回送するガード（SESSION_BOARD_DATE 明示指定時は回送しない）。reconcileは封印済みボードにno-op。
- key規約（`b:` は短slug・`[0-9a-zA-Z-]+` 準拠等）を実装前に README 語彙正本節へ書く。
- **現在時刻の注入（2026-07-09人間要望）**: ✅実装済み（2026-07-09・common.py・commit 437d48b。SS通知/初回ガイド/ミラーの3注入すべてに行内追記・既存テスト97本無変更で全緑・実地確認済み）。残タスク: `現在:` の回帰assertを test-shims.sh に追加（Turso変更のコミット後、段階2のassert更新と同時）。
- goals-summary と biz-summary の集計所有者を1つに決めてから実装（役割分担を README に明文化）。
