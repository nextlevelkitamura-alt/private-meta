# 実装の共通コンテキスト

このprogramの実装担当（implementer）が全子で共通して守ること。program.mdの流れを複製せず、役割固有の規約だけを書く。

## 対象と環境

- focusmap: `/Users/kitamuranaohiro/Private/projects/active/focusmap`（Next.js App Router / TypeScript / Tailwind / Radix UI / lucide-react。Node >=20。ローカルは必ず `http://localhost:3001`・別ポートへ逃がさない）
- session-board: `/Users/kitamuranaohiro/Private/personal-os/AIエージェント基盤/hooks-registry/shared/session-board/`（Python・runtime非依存・hook非ブロッキング）
- DB: Turso/libSQL（board=personal-os-board・inbox=personal-os-inbox）。tokenはKeychain経由で取得し、値をコード・文書・ログ・commitへ絶対に書かない

## 守る契約

- 正本線引き: 運用データ=DB／計画書・的の文面・知識=md。DBへ本文コピーを作らない
- repo選択肢の正本は `repo-registry/repo概要.md`。DBのreposマスタは参照コピー（同期方向は registry→DB のみ）
- todosスキーマは子01の定義が正本。後続子はカラム追加のみ可・破壊的変更禁止
- ai_status語彙: 未検知→検知→立案中/実行中→確認待ち→完了（確認待ち=人間承認ゲート。承認なしに完了へ飛ばさない）
- focusmapのGit運用（main直・こまめにコミット・push/本番反映は人間ゲート・検証は明示された範囲だけ）は focusmap/AGENTS.md に従う
- session-boardの3状態（🟢⏸🔵）・行形式・デイリーmd見出し文言の契約は shared/session-board/AGENTS.md と my-brain/ゴール/AGENTS.md に従う
- UIはモバイルファースト（タップ44px・幅375pxで崩れない）・Focusmap共通デザイントークン。画面の正は `../references/` のモック4枚

## テスト

- focusmap: `cd /Users/kitamuranaohiro/Private/projects/active/focusmap && npm run build`（＋計画で明示された場合のみ `npm run test:run` / 表示確認）
- session-board: `/Users/kitamuranaohiro/Private/personal-os/AIエージェント基盤/hooks-registry/shared/session-board/tests/` を全通過させる
- 完了時は各子の実行契約どおり result packet を返す

## スタイル

- 既存コードの流儀に合わせる（focusmap=既存コンポーネント・`src/lib/turso/` の流用優先・小さく分割／session-board=状態遷移はcommon.pyへ集約・shellは薄い補助だけ）
- コミットは機能単位でこまめに・日本語可。`git add -A` 禁止・自分が触ったパスだけ指定
