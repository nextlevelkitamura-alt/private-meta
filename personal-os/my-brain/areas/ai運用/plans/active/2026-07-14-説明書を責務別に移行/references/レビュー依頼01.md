# レビュー依頼01: 説明書を責務別に移行

## レビューの目的

`personal-os/説明書/` を削除したコミットが、現行の運用導線を壊さず、内容を責務を持つ正本へ一意に移せているかを判定する。

## 対象としない範囲

- 現在ワークツリーで並行中の session-board 移設・hook再編・plan-triage再設計。
- `plans/done` / `plans/archive` / `plans/paused`、過去デイリー、deleted log に残る当時の説明書パス。
- コミット `ae9bdb6` 以外の未コミット差分。

評価は必ず `git diff b60fddf..ae9bdb6` と `ae9bdb6` 時点のファイルを基準にし、現在のdirty worktreeの内容で判定を上書きしない。

## 項目別の評価観点

1. **説明書の削除と受け皿の一意性**
   - `personal-os/説明書/README.md`、`運用契約.md`、`指揮官ロースター.md` が削除されている。
   - 共通の実行規模・人間ゲート・レビューは `AIエージェント基盤/GLOBAL_AGENTS.md` §6-7、モデル・契約プランは `AIモデル一覧.md`、当日の状態は daily/session-board に分離され、同一規約の本文コピーを増やしていない。

2. **現行導線の旧参照がないこと**
   - `personal-os/AGENTS.md`、`AIエージェント基盤/skills/`、`hooks-registry/`、`loops-registry/`、active/planning計画を対象に、削除した説明書への実行時参照が残っていない。
   - 履歴ファイルのヒットはFAILにしない。ただし、activeな実行手順・起動時注入・現在の正本でヒットした場合はFAIL。

3. **朝会・モデル・状態の責務境界**
   - `skills/morning-routine/SKILL.md` がロースターや「今」の二重正本を作らず、daily/session-board と `AIモデル一覧.md` を参照する。
   - `AIモデル一覧.md` に金額・残量・特定日の担当・過去の編成を持ち込んでいない。

4. **実行時の互換性**
   - session-board の注入文、plan-triage / inbox-triage、cockpit関連の参照が新しい正本を指す。
   - patchが並行変更を巻き込まず、削除済み文書を読み込む実行導線を残していない。

5. **計画・履歴・検証の整合**
   - `plan.md` の完了条件・実装結果・検証結果・次のゲートが実コミットと一致する。
   - `ai運用/決定ログ.md` に廃止理由、新しい正本、履歴を変更しない範囲が記録されている。
   - diffの空白エラー、対象スクリプトの構文、関連テストに矛盾がない。

## 実施方法

1. `git show --stat ae9bdb6` と `git diff b60fddf..ae9bdb6` を読み、対象外の差分を含めないことを確認する。
2. 上の各観点を、対象ファイルと行・節を根拠にPASS/FAIL/判定不能で採点する。
3. 必要なら、コミット内容だけを対象に静的検索・構文検査・既存テストをread-onlyで実行する。
4. 変更・stage・commit・pushはしない。

## 報告形式

- 重大度順に、`P0/P1/P2`、対象ファイル、根拠、期待状態を記載する。
- 指摘なしの場合も、5項目すべての根拠を短く示して `PASS` と明記する。
- 評価者はファイルを編集しない。評価結果の正式記録 `評価01.md` は指揮官が作成する。
