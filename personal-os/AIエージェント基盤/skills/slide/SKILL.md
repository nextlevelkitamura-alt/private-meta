# /slide — スライド作成スキル

スライド・プレゼン資料を作るためのグローバルスキル。
**ライト / しっかり** の2モード。テンプレを選んで効率化、選ばない要素は対話で詰める。

## 起動時に必ずやる2つ

### 1. 起動時読込（必須）
- `memory/slide.md` — 過去デッキ・ブランド・ノートブックインデックス
- `テンプレ/ブランド/_last-used.txt` — 最後に使ったブランド名
- `テンプレ/ブランド/*/brand.yaml` — ブランド一覧（名前だけ把握）
- `~/Private/副業/素材/wiki/` の存在確認 — 素材ディレクトリのインデックス把握（中身は必要時に Read/Grep）

### 2. モード選択（最初のラリー）

```
スライドを作ります。どのモードで進めますか？

A) ライト    — 4ラリーでサクッと作る（社内共有・個人資料・速報など）
B) しっかり  — 7問ロジックツリーで深掘り（営業提案・登壇・社外提出）
C) 案件型    — 案件/求人/キャリア/営業/LPを1枚ずつ壁打ちで詰める（Codex=画像生成/Claude=プロンプト化）
```

**自動推奨しない**。ユーザーに選ばせる。

## モード分岐

| 選択 | 進む先 |
|---|---|
| A) ライト | `light.md` を読み、4ラリーフローを実行 |
| B) しっかり | `full.md` を読み、7問ロジックツリーを実行 |
| C) 案件型 | `references/project-deck-mode.md` を読み、1枚ずつ壁打ち＋画像環境分岐で進める（旧 project-slide-workflow を吸収） |

## 共通の前提

### 素材は `~/Private/副業/素材/` から引く（ローカル git 管理）

スライド生成のソースは原則ここから取る。NotebookLM のようなクラウドに依存せず、git管理されたローカル md を Claude が直接 Read/Grep する。

```
~/Private/副業/素材/
├── raw/           素材投下先（メモ・URL・文字起こし等）
├── wiki/          アトミックノート（再利用可能形式）
│   ├── 体験/      個人体験・失敗談・成功談
│   ├── スタンス/  思想・価値観・アンチテーゼ
│   ├── パターン/  営業フック・差別化フレーム
│   ├── データ/    数値・統計・調査結果
│   └── 引用/      書籍・記事・他者発言
└── decks/         スライド成果物（最終版・再利用候補）
```

詳細は `~/Private/副業/素材/README.md` 参照。

### 情報源の選択肢（必ず聞く）

```
情報源どうしますか？（複数選択可）
A) 素材/wiki/ から引く（ローカル既存ナレッジ活用）
B) ネット情報をリサーチ官に集めさせる
C) 添付資料あり（パス指定）
D) 追加で聞き出す
E) 今ある情報で十分
```

### 生成エンジンの選択肢（必ず聞く・自動推奨しない）

内容が固まったあと、最後に**どこで生成するか**を選ぶ：

```
生成エンジンを選んでください:
A) Claude for PowerPoint アドイン
   - スライドマスター・配色・フォント自動踏襲
   - Instructions永続設定、外部提出向き
   - Claude サブスク + PowerPoint 365 デスクトップ版必須

B) NotebookLM ルート
   - 素材を投入してから生成
   - 音声化（ポッドキャスト）も可能
   - notebooklm-mcp-cli インストール済み（uv tool install）
```

詳細は `references/powerpoint-route.md` / `references/notebooklm-route.md`。

### 画像生成の選択肢（必要時に毎回聞く・自動推奨しない）

スライドに画像が必要になったら、**毎回必ず聞く**:

```
画像どうしますか？
A) /generate-images スキルを呼ぶ（対話で揃える・推奨）
B) 画像生成APIを直叩き（gpt-image / Imagen / Gemini）
C) 添付済みの画像を使う（パス指定）
D) 画像なしで進める
```

`/generate-images` は別グローバルスキル。Codex `image_gen` で生成する。

## テンプレ4カテゴリ（外部管理）

スライド生成の素材として、以下を `テンプレ/` 配下で管理。各モードから選択して使う。

| カテゴリ | 場所 | 内容 |
|---|---|---|
| 構成 | `テンプレ/構成/` | ストーリー型7種（PASTOR / ベネフィット / 圧縮 / 営業提案 / 講演登壇 / 自己紹介 / 社内共有） |
| レイアウト | `テンプレ/レイアウト/` | 見た目パッケージ8種（右画像-左文字 / 見出し本文中心 / 図解中心 / 全画面写真 / 3カラム比較 / タイムライン / データ表-グラフ大 / ミニマル文字のみ） |
| 差別化 | `テンプレ/差別化/` | 「自分のもの」にする問い5種（独自視点 / 体験ストーリー / スタンス表明 / 数値の独自解釈 / アンチテーゼ） |
| ブランド | `テンプレ/ブランド/` | 色・フォント・ロゴ。`_last-used.txt` で最後使用を記録 |

**選ばない要素は必ず対話で個別に聞く**（テンプレなしで進めない）。

## エージェントチーム

- **指揮官（メイン）**: 対話・記憶読込・モード判定・全体統括・各種CLI実行
- **企画官（Agent）**: ストーリーライン3案・全体ボリューム設計・PPT/NLM プロンプト合成
- **リサーチ官（Agent）**: ネット情報収集・構造化・ソース投入順設計

呼び出しの判断:
- フル生成（しっかり） → 企画官・リサーチ官を必要に応じて呼ぶ
- ライト → 指揮官だけで完結（リサーチが必要なときのみリサーチ官）

詳細プロンプトは `references/企画官-prompt.md` / `references/リサーチ官-prompt.md`。

## 4段階フロー（素材エコシステム）

スライド作成は単発で終わらせず、素材エコシステムに組み込む：

```
[Capture] 素材を ~/Private/副業/素材/raw/ に即投下（普段の生活で）
   ↓
[Compile] 週1で raw/ → wiki/ アトミックノート化（手動 or 別スキル）
   ↓
[Compose] /slide 起動時、wiki/ を grep/Read で引いてスライド生成 ← ココ
   ↓
[Archive] 完成スライドの強い素材（キャッチコピー・図解・検証済み数字）を wiki/ に逆流
```

slide スキルが担うのは [Compose] フェーズ。完了時に [Archive] を必ず実行する：
- 完成 .pptx を `~/Private/副業/素材/decks/{資料名}.pptx` にコピー
- 強い素材を `~/Private/副業/素材/wiki/{カテゴリ}/` に書き戻し（ユーザー合意の上）

## スプシログ（スライド管理スプレッドシート）

Row 4: [Primary][#hex]   [Secondary][#hex]  [Accent][#hex]
Row 5: [Font見出し][value] [Font本文][value]
Row 6: [構成テンプレ][value] [レイアウト][value] [差別化][value]
Row 7: (空白)
Row 8: ● プロンプト
Row 9: [Instructions][長文テキスト]
Row 10: [本文プロンプト][長文テキスト]
Row 11: (空白)
Row 12: ● スライド構成  （← 列方向にページが横展開）
Row 13: [ページ][1][2][3][4]...
Row 14: [タイトル][t1][t2][t3]...

### 書き込みタイミング（随時）

| タイミング | 書き込み内容 |
|---|---|
| 資料名・モード確定時 | タブ作成 + Row 1-2 |
| ブランド・色味確定時 | Row 3-6 |
| プロンプト確定時 | Row 8-10 |
| スライド構成確定時 | Row 12-17 |

### gws コマンド

```bash
SHEET_ID="1z_inGsz2plWVxSmyNfyq5P0uplS0uPM-QulQYEEnvm4"
TAB="2026-05-08 資料名"

# タブ作成
gws sheets spreadsheets batchUpdate \
  --params "{\"spreadsheetId\": \"$SHEET_ID\"}" \
  --json "{\"requests\": [{\"addSheet\": {\"properties\": {\"title\": \"$TAB\"}}}]}"

# 値の書き込み（例: Row 1-2）
gws sheets spreadsheets values batchUpdate \
  --params "{\"spreadsheetId\": \"$SHEET_ID\"}" \
  --json "{
    \"valueInputOption\": \"USER_ENTERED\",
    \"data\": [{
      \"range\": \"'$TAB'!A1:F2\",
      \"values\": [
        [\"資料名\", \"副業スタートガイド\", \"モード\", \"しっかり\", \"エンジン\", \"PPT\"],
        [\"生成日\", \"2026-05-08\", \"ブランド\", \"nextlevel\", \"\", \"\"]
      ]
    }]
  }"

# 値の更新（例: 色味 Row 4-5）
gws sheets spreadsheets values batchUpdate \
  --params "{\"spreadsheetId\": \"$SHEET_ID\"}" \
  --json "{
    \"valueInputOption\": \"USER_ENTERED\",
    \"data\": [
      {\"range\": \"'$TAB'!A3\", \"values\": [[\"● 色味・デザイン\"]]},
      {\"range\": \"'$TAB'!A4:F4\", \"values\": [[\"Primary\", \"#1A1A2E\", \"Secondary\", \"#16213E\", \"Accent\", \"#0F3460\"]]},
      {\"range\": \"'$TAB'!A5:D5\", \"values\": [[\"Font見出し\", \"Noto Sans JP Bold\", \"Font本文\", \"Noto Sans JP\"]]}
    ]
  }"
```

**ルール**: タブ作成後、各情報が確定するたびに即書き込む（後でまとめて書かない）。

## 重要な原則

1. **モード選択は必ず対話で**: ライト/しっかりを自動判定しない。最初のラリーで聞く。
2. **テンプレを選ばない要素は対話で深掘り**: 構成・レイアウト・差別化・ブランドのうち「使わない」「カスタムで」と言われた要素は個別に聞く。
3. **エンジン・画像生成・情報源は毎回聞く**: 自動推奨しない。
4. **重要度を見極める**: ライト選択でも営業/提案/外部公開なら3問深掘りを追加。
5. **ブランドは記憶層に**: 一度作ったら `テンプレ/ブランド/{name}/` で永続化、`_last-used.txt` で次回デフォルト。
6. **素材はローカル git 管理**: `~/Private/副業/素材/` が真実の源泉。NotebookLM 等のクラウドに依存しない。
7. **完成後の Archive を忘れない**: 強い素材は wiki/ に書き戻して再利用源泉化。
8. **スプシログは随時書く**: 資料名確定→タブ作成、各情報確定→即書き込み。まとめ書き禁止。
