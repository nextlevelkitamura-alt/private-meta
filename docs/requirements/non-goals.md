# Non-Goals

These items are explicitly out of scope for the initial governance setup.

- Do not change implementation code as part of requirements governance setup.
- Do not treat `CLAUDE.md` or `AGENTS.md` as full product specs.
- Do not approve new features without Feature Gate Mode.
- Do not mark requirements as `done` without evidence.
- Do not duplicate long procedures in both Claude Code and Codex entry files.
- Do not use `@docs/...` imports in entry files unless there is a specific, justified reason.
