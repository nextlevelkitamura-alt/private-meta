session-start の実行フロー

開始時は「通知だけ」。ボードの行を作るのは、次の意味ある入力が来た時。

SessionStart が発火
reconcile-and-notify.py --runtime claude|codex が stdin JSON を受け取る。
生存照合
古い🟢/🔵行を実体トランスクリプトで照合し、必要なら⏸へ直す。
キーだけを通知
この時点ではボード行を作らない。補助セッションを🟢の幽霊枠にしないため。
最初の意味ある入力
../prompt-register/register-and-guide.py が行を登録し、開始ガイドをAIへ注入する。
AIが一度だけ整える
依頼を理解した後、目標・種別・今・計画・モデルを board.py update で正す。

出力だけruntimeで違う

runtime引数返すもの
Claude--runtime claudeplain text の開始通知
Codex--runtime codexadditionalContext を含む JSON

状態ロジックは共通の shared/session-board/common.py にあり、このファイルは出力形式だけを選ぶ。

登録表はruntimeごとに違う

Claudeは ~/.claude/settings.json の hooks 項目から、Codexはrepoの codex/hooks.json から、この共通Pythonを呼ぶ。Claude用の同期ファイルや別Pythonは置かない。
