# 外出先からMacフォルダを参照する選択肢

更新: 2026-07-13

## 結論

未コミットを含むMac上の現物を外出先で読むなら、第一候補は **Files + SMB + Tailscale**。ファイル内容の確認や作業継続まで必要なら Codex Remote、完成資料だけを常時持ち歩くなら iCloud Drive、push済み履歴を見るなら GitHub Mobile を使う。

## 使い分け

| 目的 | 手段 | 見える範囲 | 注意 |
|---|---|---|---|
| Macの今のフォルダを読む | Files + SMB + Tailscale | 未push・未コミットを含む現物 | Macが起動・接続中であること |
| 内容・差分を調べて作業も続ける | Codex Remote | 接続したMac上の作業環境 | Finderの代替UIではない |
| 定番資料をすぐ読む | iCloud Drive | 同期したコピー | Git作業ツリーを同期しない |
| 履歴や共有済みの状態を見る | GitHub Mobile | push済みのスナップショット | 現在のローカル状態は見えない |

## 推奨構成

1. 日常の現物閲覧は、共有範囲を絞ったSMBをTailscale経由でFilesから読み取り専用で開く。
2. よく読むMarkdown・PDF・画像だけはiCloud Driveへ選抜コピーする。
3. 調査・差分確認・作業再開が必要な時だけCodex Remoteを使う。
4. GitHubは履歴・共有のために使い、モバイル閲覧だけを目的に高頻度pushしない。

## 安全上の境界

- 作業中のGitリポジトリや `.git` をクラウド同期の対象にしない。
- SMBをインターネットへ直接公開せず、共有フォルダ・ユーザー・権限を最小化してTailscale経由にする。
- 高頻度pushはpush自体の従量課金より、Actions起動・LFS・履歴肥大化を確認して判断する。
