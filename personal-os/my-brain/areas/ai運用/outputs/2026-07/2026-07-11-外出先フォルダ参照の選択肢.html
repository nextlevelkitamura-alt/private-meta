外出先で「いまのMac内」を見る方法

GitHubは速い閲覧先だが、未プッシュの状態は見えない。実ファイルを読むなら Files＋SMB/VPN、ファイルを調べたり作業を続けるなら ChatGPT/Codex Remote、固定資料だけなら iCloud Drive が役割に合う。

まず結論

Files＋SMB＋Tailscale 第一候補

未プッシュのフォルダをそのまま閲覧する用途。iPhoneのFilesから、Macで共有した指定フォルダを読み取り専用で開く。

ChatGPT / Codex Remote 作業もする

Mac上の現物をAIに調べさせる用途。差分・ログ・ファイル内容を質問し、必要なら作業も継続できる。Finderの代替ではない。

iCloud Drive＋Files 固定資料向け

よく見るMarkdown・PDF・画像だけを同期する用途。Gitの作業ツリーではなく、モバイル閲覧用のコピーを置く。

GitHub Mobile 履歴・共有向け

push済みのスナップショットを最速で見る用途。コード検索・リポジトリ閲覧には強いが、Macの未保存・未push状態は見えない。

「どこにあるファイルか」で分ける

一つの仕組みに寄せず、ローカルの正本・モバイル用コピー・Git履歴を分離すると、速さと安全性が両立する。

おすすめ構成は二層

日常はFiles＋SMB/VPN、AIに探させたい時だけCodex Remoteにする。GitHubは「履歴を残す」目的に戻す。

層A：現物を即座に見る

Macで必要な親フォルダだけを共有し、Filesアプリから smb://<MacのTailscale IP> へ接続する。Mac側は共有フォルダとユーザーを限定し、iPhoneでは読むだけにする。

層B：いつもの資料を軽く見る

閲覧頻度の高い資料だけをiCloud Driveへ。作業中のGitリポジトリや .git をクラウド同期の対象にせず、README・計画・PDF・画像などの選抜コピーに留める。

最小セットアップ

外出先で読むだけなら、Macをインターネットへ直接公開せず、VPN/メッシュ経由にする。

  1. 1
    Macで共有対象を1フォルダに絞る

    macOSの「一般 ＞ 共有 ＞ ファイル共有」で、必要なフォルダだけを追加し、自分のアカウントを「読み出しのみ」にする。

  2. 2
    MacとiPhoneにTailscaleを入れる

    同じtailnetへ参加させる。外部公開ではなく、Tailscale経由でMacのSMB共有に届くようにする。

  3. 3
    iPhoneのFilesで「サーバへ接続」

    smb://<MacのTailscale IP> を登録する。Filesには最近使ったサーバが残るので、次回から数タップで開ける。

  4. 4
    補助としてCodex Remoteをペアリング

    ChatGPTデスクトップアプリが起動したMacを、モバイルのRemoteから接続する。ファイル内容・差分・ログを「これを探して」と聞く用途に使う。

10分ごとのGitHub pushに料金は掛かるか

通常のGit pushそのものに「1回いくら」という課金はない。ただし、pushで起動するActions・LFS・大きく増え続ける履歴には別のコストと運用負債がある。

直接のpush料金 原則なし

GitHubの公式課金項目では、通常のGit push回数は従量メーターになっていない。GitHub Freeで10分おきにpushしても、push回数そのものの課金は発生しないという整理でよい。

GitHub Actions 最重要の例外

on: push のworkflowがあれば、変更ごとに実行される。非公開リポジトリのGitHub Freeは標準runnerが月2,000分・artifact 500 MB・cache 10 GBまで。上限超過後に予算を許可していれば課金対象になる。

Git LFSと大容量ファイル 動画・画像・DBに注意

LFSはFreeで保存・帯域とも各10 GiB/月が含まれるが、更新のたびにファイル全体が新バージョンとして計上される。通常Gitも履歴が積み上がり、GitHubはリポジトリを1 GB未満、少なくとも5 GB未満を強く推奨している。

履歴の読みやすさ 料金以外の問題

最大で 144回/日・約4,320コミット/月。本番ブランチをこの用途にすると履歴と通知が荒れる。必要なら mobile-snapshot のような専用ブランチを使い、Actionsはそのブランチを除外する。

避けたい構成

Gitの作業ツリー全体をiCloud/Dropboxで常時同期しない

クラウド同期は「モバイルで読む選抜資料」には便利だが、作業中のリポジトリには多数の更新頻度が高い管理ファイルがある。同期衝突時の復旧を増やさないため、正本の作業ツリーはMacローカルに置き、共有はSMB/VPNまたは選抜コピーで行う。

判断

「今のフォルダを眺めたい」

Files＋SMB＋Tailscale。push不要で、未コミットも含めてそのまま見える。Macが起きてネット接続されていることだけが前提。

「このファイルの要点や差分を知りたい」

Codex Remote。モバイルからMac上のタスクを動かし、質問・確認・承認・作業継続までできる。フォルダを手で巡回するUIではなく、作業エージェントとして使う。

「閲覧中心の資料を即開きたい」

iCloud Driveのモバイル閲覧用フォルダ。接続待ちがなく、Macが寝ていても開ける。正本ではなくコピーと割り切る。

根拠・出典

  • OpenAI — Remote connections：モバイルから接続したMacのローカル文書・ファイル・ツールにアクセスできること、Macが起きてネット接続されている必要があることを確認。
  • Apple — iPhoneのFilesでサーバへ接続、Macのファイル共有：iPhoneからサーバへ接続し、Mac側で共有フォルダと権限を限定できることを確認。
  • Tailscale — iPhoneからmacOSのSMB共有へ接続：FilesからMacのTailscale IPを入力して共有フォルダを閲覧する手順を確認。
  • Apple — DesktopとDocumentsをiCloud Driveへ：iCloud Driveは端末間でファイルを同期し、反映時間は接続状況・回線に依存することを確認。
  • GitHub — Actions billing、Git LFS billing、large files：課金対象・Free枠・大容量リポジトリの注意点を確認。push単体の従量課金が明記されていない点は、これらの公式課金体系からの判断。
  • GitHub — GitHub Mobile、pushで起動するActions：モバイルでのリポジトリ閲覧と、pushがworkflowを起動し得ることを確認。

注：料金・機能・提供範囲は2026-07-11時点の公式ドキュメントを確認。Tailscaleのプラン料金は本調査の対象外。
