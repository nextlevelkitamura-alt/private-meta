# session-start — 開始時の宣言手順

セッション開始の流れ（2026-07-08 再設計 / 幽霊枠掃除で更新）:
SessionStartフックは生存照合とキーの1行通知だけを行う（**枠は登録しない**）。
最初の意味あるプロンプトで UserPromptSubmit フックが行の**枠**（時刻・🟢・repo・runtime）を機械登録し、
同時に**このガイド相当の注入**（記入コマンド・種別定義・既存目標一覧・計画チェーン。生成は common.py・目標が未記入の間だけ）を行い、
「今」列にプロンプト先頭24字を初回だけ仮置きする。**以降の意味づけはAIの仕事**。
（枠登録を初回プロンプトへ寄せたのは、プロンプトを持たない補助セッションが🟢の幽霊枠として残るのを防ぐため）

`<board>` = `~/Private/personal-os/AIエージェント基盤/hooks-registry/hooks/session-board/board.py`。キーは注入メッセージに記載。

## エージェントの仕事

1. **開始**: 最初の意味ある依頼を理解した直後に1回、行を正す:
   `<board> update --key <キー> --type <計画|実装|リサーチ|レビュー|その他> --goal "<達成したらこのセッションを閉じられる1行・30字以内>" --now "<いま着手する一歩・20字以内>" --plan "<企画名[/NN] か なし>" --model <自分のモデル名 例: fable5>`
   - 種別の定義（5種・一言＋成果物）の正本は `README.md`。迷ったら「その他」でよく、後で直すのが正常。
   - **計画列**（`--plan`・開始時に記入）: 実装・レビューは拠り所の計画、計画種別はこれから置く先を書く。①1〜2ファイル ②容易に戻せる ③人間ゲート無し が全YESのサクッと作業なら `--plan なし`。値の語彙（`?`／`なし`／短縮参照 `企画名[/NN]`・`ai運用:企画名[/NN]`）の正本は `README.md`。
   - 注入された「いま動いている他の目標」に同じ目的があれば、**その文言をそのまま --goal にコピー**して合流する
     （親名=目標名で、複数チャットの成果が「終わったこと」の1つの親に集まる）。
2. **作業が進んだら**: 「今」がズレたら `update --now`。目的・種別が変われば `--goal`／`--type`（開始時刻とキーは固定）。
3. **節目（一区切り・機能完成）**: 「終わったこと」に子を入れ子で積む:
   `<board> log --key <キー> --repo <repo> --parent <目標名> --entry <成果1行>`
   → 時刻と所要 `(+Nm)` は自動付与（推測で書かない）。行は消えず🟢のまま続行。親は重複せず最新順に積まれる。
4. **バックグラウンドのサブエージェントを使う時**: SubagentStart/SubagentStop 受け口が登録済みの環境では
   `sub-start`/`sub-end` が体数（`↳ 🔵 サブN体`）ごと自動増減するので手動操作は不要。
   未登録の環境の手動フォールバック: 起動したら `<board> flip --key <キー> --state sub`（🔵）、
   サブ完了通知で戻ったら `flip --state run`。
5. **計画種別の置き場**: `repo-registry/repo概要.md` で担当repoだけを判定（**cwdでなく依頼の中身で**）→
   そのrepoの最寄り `AGENTS.md` で領域・プロジェクト・計画箱を解決 → 宣言された検索範囲で既存planを確認する。
   既存planがあれば合流し、無ければrepoが宣言した箱へ作る。registryに担当repoが無い、repo `AGENTS.md` が無い、
   計画箱が未宣言・複数候補で曖昧、既存planが複数競合する時は、root `plans/` を推定・作成せず停止して人間に確認する。
6. **Privateから対象repoへの書込み**: canonical repo path・plan参照・worktree cwd・許可path・開始時Git snapshotを引き継ぎ、
   対象repoをrootとする新しい可視sessionを起動する。既存session IDの移管・reparentはしない。新sessionのsession-board登録と
   対象repo `AGENTS.md` の読込みを確認後、Private側は引継ぎ完了としてfinishする。調整役として残す場合だけ2行併存を許し、
   両行の役割と終了責任を明記する。

## 登録しないもの

- subagent／headless（`AIJOBS_RUN`）／スラッシュコマンド／空・添付のみ。

## してはいけないこと

- 他セッションの行に触れる／完全なsession id・SHA・secretを書く。
- 入れ子ブロック（`<!-- goals-summary -->`〜）の派生行（`### 本日の目標`・目標親行・`↳ サブN体`）を
  手で編集する（自動再描画で消える）。セッション行の変更は必ず `board.py` 経由（sub:N も手で書かない）。
