session-board hooks · 07-13

セッション開始時に注入されるフックと、その参照マークダウン

このセッションで実際に発火したのは SessionStart の「キー通知1行」と UserPromptSubmit の「フルガイド」。生成元は common.py の3関数、参照マークダウンは session-board/ フォルダ。原則は Python＝枠と機械処理／AI＝意味づけ。

登録マップ ① SessionStart ② UserPromptSubmit ③ Stop 参照md 出典

要点

① SessionStart

キー通知1行だけ。生存照合（reconcile）を回し、ボードキーを知らせる。行（枠）はまだ作らない——プロンプトを持たない補助セッションが幽霊枠になるのを防ぐため。

② UserPromptSubmit

枠登録＋二段注入。初回プロンプトで行を登録し、目標未記入＝フルガイド／記入済み＝2〜3行ミラーを注入。今回のセッションはフルガイド版。

③ Stop

締めの網。run→⏸へ機械flip（毎ターン）。加えてprompt型フックが「大目標達成＋満足の気配」のときだけ完了手順を促す（節目のみ）。

settings.json のフック登録マップ

~/.claude/settings.json に登録された session-board の受け口。実体はすべて ~/.claude/agent-hooks/（→ repo の hooks-registry/claude/ への symlink 窓）越しに解決される。

※同イベントにもう1本 orca の claude-hook.sh グルーが並ぶ（session-board とは別系統）。

Stop2本立て

  • session-board-session-end.py（command型）: run→⏸ 機械flip＋reconcile
  • milestone 判定（prompt型・Claude専用）: 節目でのみ完了手順を促す

SubagentStart / SubagentStop

session-board-subagent.py（両イベント同一）。サブエージェント体数を sub-start／sub-end で自動増減し 🔵 を上げ下げ。

ガード（登録・作用しない条件）を見る

AIJOBS_RUN 非空（headless）／ session_id が agent-*（subagent）／ transcript が */subagents/*／ スラッシュコマンド・空・添付のみのプロンプト。これらは記録も注入もされない（ad-hoc な対話だけ拾う）。

③ Stop の2本立て（機械flip ＋ 節目prompt）

Stop は毎ターン走る command型（状態を⏸へ倒す）と、節目だけ効く prompt型（完了手順を促す・Claude専用）の2本。後者は「大目標達成＋満足の気配」のときだけ ok:false を返して session-end.md の手順を注入する。

command型 session-end.py

run のときだけ ⏸ へ機械flip＋reconcile。sub／wait は触らない。ブロックしない（毎ターン・完了確認とは独立）。

prompt型 milestone 判定

会話を読み、完了＋満足を判定。継続中・確認済み・締め済みなら ok:true（素通し）。大目標達成＋満足の気配のときだけ ok:false で完了手順を促す。迷ったら true 側（聞かなさすぎに倒す）。

prompt型フックの全文（settings.json の Stop prompt）を見る
このセッションの会話を読み、開始時の大目標が達成されユーザーが満足した気配があるかを判定せよ。JSONだけを返す。 - まだ作業継続中／直前の1往復が済んだだけ／依頼の途中 → {"ok": true} （＝普通に停止してよい。人間の次の指示を待つ。確認は出さない） - 完了確認を直近で既に提示済み（人間の返答待ち）、または 完了報告・git仕上げ（記載/コミット/push）を既に実行済み → {"ok": true} （＝同じ節目で確認・締めを二重にしない。返答待ち／完了後の再点火ループを避ける） - 大目標（例: サイト制作・機能実装・調査・設計）が一通り達成され、かつユーザーが満足の気配 （修正要求が止まった／肯定的反応／「ありがとう」「いいね」「OK」等の締めの気配）→ {"ok": false, "reason": "session-board の完了報告手順（…/session-board/session-end.md）を実行せよ。完了判断→人間に①記載②コミット③マージ&push④続けるを確認→OKならboard.py finish＋git仕上げ。"} 迷ったら {"ok": true}（未達側に倒し、人間に委ねる）。1往復ごとの細かい完了では false にしない。
