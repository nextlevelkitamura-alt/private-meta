hooks-registry 再編と symlink 露出

Claude と Codex が共有するイベント実装を、1つの正本として理解・保守できる状態にするための実行プログラムです。詳細な実行契約は同階層の program.md が正本です。

決定済み: Claude のフック登録は ~/.claude/settings.json に直接置く。設定ファイル全体を symlink にはしない。登録を同期するためだけの apply-hooks.py と repo 内の Claude 用 hooks.json は廃止する。

完成後に見える構造

イベント実装hooks-registry/events/<event>/。実行する Python と、同名の処理説明 Markdown を並べる。
共有エンジンhooks-registry/shared/session-board/。イベント実装から使う共通ロジックを置く。
Claude の登録~/.claude/settings.json。repo に置くのは実行物と説明だけで、設定全体の正本にはしない。
Codex の登録hooks-registry/codex/hooks.json を正本にし、~/.codex/hooks.json から symlink 露出する。
説明書各フォルダの AGENTS.md が正本。人間向けの図は対応する AGENTS.html または本ページ。

最終的に何がどうなるか

日常的に見る場所は hooks-registry/events/ です。ここを見れば、イベントごとに「いつ」「どの Python を」「何のために」動かすかが分かります。Claude と Codex は、その同じ実装を呼びます。

イベント実装は 1 つ、入口だけ runtime ごとに異なる
① この repo を編集する

events/session-start/reconcile-and-notify.py と、同名の .md を見る。ほかのイベントも同じ形。

→
② Claude / Codex が呼ぶ

Claude は settings.json、Codex は hooks.json に「この Python を呼ぶ」とだけ登録する。

→
③ session-board が更新される

共通エンジン shared/session-board/ が、セッション状態と Daily 実行ログを扱う。

Claude の設定は 1 ファイルのまま

~/.claude/settings.json には hooks だけでなく model・permissions なども同居する。これは Claude の仕様であり、分割や symlink はしない。

→
Codex の hooks は repo を正本にする

hooks-registry/codex/hooks.json を ~/.codex/hooks.json に symlink 露出する。

→
二重管理をなくす

Claude 用の apply-hooks.py と repo 内 Claude hooks.json はなくなる。設定を複製して同期する作業もない。

完了の定義

  • Claude は settings.json の直接登録だけで対象イベントを起動できる。
  • Claude 用の apply-hooks.py と repo 内の hooks.json は残っていない。
  • イベントごとの「何を実行するか」が、フォルダ名・Python 名・同名 Markdown・AGENTS.md から追える。
  • Claude / Codex の runtime 窓と Codex の登録正本が、読み取り専用診断とテストで確認できる。
  • 古い milestone / Orca 前提や、実態と異なる説明が残っていない。
