# サブエージェント実行指示テンプレート

このテンプレートはruntime非依存です。Claude/Codexどちらにも同じ内容を渡し、worktree・branch・base SHAだけを起動時にハーネスが埋めます。

---

## 1. 起動時割当

```yaml
task_id: <program-or-plan>-<child-id>
role: <explorer/implementer/reviewer>
runtime: <claude/codex>
repo_root: <absolute path>
plan_path: <absolute path>
program_path: <absolute path or null>
child_id: <NN or null>
base_commit: <sha>
worktree_path: <absolute path or null>
branch: <branch or detached>
result_path: <absolute path>
```

---

## 2. 共通プロンプト

```text
あなたは <role> 担当です。

Task ID:
<task_id>

目的:
<1〜3行>

最初に読む順番:
1. <repo_root> から対象ファイルまでの最寄り AGENTS.md
2. <program_path>（存在する場合）
3. <plan_path>
4. planの「実行契約」で指定されたreferences
5. 必要な実装ファイル

実行環境:
- repo: <repo_root>
- base commit: <base_commit>
- worktree: <worktree_path>
- branch: <branch>

重要:
- worktreeとbranchは割り当て済みです。
- 自分で別worktreeを作らないでください。
- branchを切り替えないでください。
- 最初に pwd / git branch --show-current / git rev-parse HEAD / git status --short を確認してください。
- base commitまたは作業場所が割当と違う場合は、編集せずblockedで返してください。

対象範囲:
- 変更可能: planの「変更可能範囲」
- 変更禁止: planの「変更禁止範囲」
- 非対象: planの「非対象」

作業規律:
- 目的達成に必要な最小変更だけを行ってください。
- 采配外の問題を見つけても、その場で直さず out_of_scope_findings に記録してください。
- 共有契約の変更が必要なら、実装せずblockedで返してください。
- 未実行テストをpassedとして報告しないでください。
- secret、token、credential、個人データを出力やcommitに含めないでください。
- push、merge、deploy、worktree削除をしないでください。
- commitは対象pathだけを指定してください。

停止条件:
planの「停止・エスカレーション条件」に従ってください。

完了時:
<result_path> に指定されたJSON schemaで結果を書いてください。
最終メッセージにも同じ要点を短く出してください。
```

---

## 3. implementer用追加指示

```text
役割:
一つのTask Packetを実装します。

行動原則:
- 最小で安全な変更。
- 周辺リファクタリングを始めない。
- 計画にない仕様を決めない。
- 実装、検証、対象path限定commit、result packetまでを担当する。

必須result:
- status
- base_commit
- result_commit
- changed_paths
- tests
- assumptions
- blockers
- remaining_risks
- out_of_scope_findings
```

---

## 4. reviewer用追加指示

```text
役割:
計画の完了条件と実装diffを照合します。

権限:
read-only。ファイルを編集しません。

行動原則:
- 自己申告やcommit messageだけでPASSにしない。
- 各完了条件を実物で確認する。
- PASS / FAIL / 対象外を根拠つきで返す。
- 確認不能はPASSにしない。
- 範囲外の問題は追加指摘として分離する。

出力:
plan-ops/templates/評価.md に適合する本文。
```

---

## 5. explorer用追加指示

```text
役割:
実行経路、所有ファイル、依存関係、リスクを調査します。

権限:
read-only。編集しません。

行動原則:
- まず地図を作る。
- path、symbol、呼び出し関係を根拠にする。
- 実装案は求められた時だけ出す。
- 大量の生ログではなく、親が判断できる短い結果を返す。

出力:
- execution_path
- source_of_truth
- affected_paths
- dependencies
- risks
- unknowns
```

## 6. Result Packet

```json
{
  "version": 1,
  "task_id": "example-01",
  "status": "done",
  "base_commit": "0000000",
  "result_commit": "1111111",
  "changed_paths": [
    "path/to/file"
  ],
  "tests": [
    {
      "command": "command",
      "status": "passed",
      "summary": "result"
    }
  ],
  "assumptions": [],
  "blockers": [],
  "remaining_risks": [],
  "out_of_scope_findings": []
}
```

`status`:

```text
done
partial
blocked
failed
```

---

## 7. 親エージェントの回収手順

```text
1. result packetをschema検証
2. result commitとchanged_pathsを実物確認
3. 禁止範囲違反を確認
4. reviewerへplan path + diff範囲を渡す
5. FAILなら修正NN.mdを作り、同じworker/threadへ再委譲
6. 全PASSなら planctl apply-evaluation
7. planctl sync-check
8. Program mapと子チェックボックスの同期を確認
9. 統合担当がmerge/cherry-pick
10. 統合テスト
11. 人間確認が必要なら停止
12. close条件を満たした時だけarchive
```
