# mode-research — 競合・トレンド調査モード

「競合調査」「トレンド調査」「Threads調査」「インフルエンサー調査」「スクショしてスプシ」「投稿構造を調べて」「AIコーチングの動向」で起動。

外部の Threads / Instagram / X などを観察し、発信者・投稿・見え方・CTA・スクショをスプレッドシートに整理して、自アカウントのネタ帳・投稿型へ転用するための調査モード。

---

## 前提

- 調査は読み取り中心。スプシ作成・追記・Driveアップロード・ネタ帳追加は副作用なので、必ず dry-run / 内容提示 / ユーザー確認後に実行する。
- 投稿本文や画像は参考分析に留め、丸写ししない。転用時は「構造」「視点」「CTAの流れ」だけを抽出する。
- ログイン済みブラウザや Cookie を使う場合、認証情報・Cookie・トークンを出力しない。
- 調査対象が Threads の場合、未ログインWebFetchだけで取得できるとは限らない。取得可否を最初に小さく検証する。

---

## Step 1: 調査テーマ確定

発話から自動推定できる場合はスキップ。曖昧なら AskUserQuestion で選ぶ。

```
何を調べますか？

A. AIコーチング / AI伴走
B. AI副業 / プロンプト販売
C. AI自動化 / Claude Code / 業務効率化
D. 自由テーマ
```

AIコーチング / AI伴走の初期検索語:

- `AI伴走`
- `AIコーチ`
- `AI活用 相談`
- `ChatGPT 伴走`
- `AI副業 伴走`
- `AI初心者 サポート`
- `Claude Code 使い方`
- `AIで時短`

## Step 2: 調査深度を選ぶ

```
どの深さで進めますか？

A. クイック調査
   — Web検索 + 公開ページ確認で候補を10件前後に絞る
B. 標準調査
   — Threads検索/プロフィール確認 + スクショ + スプシ下書き
C. 深掘り調査
   — 発信者DB/投稿DB/構造分析/勝ちパターン/自アカウント転用案まで作る
```

推奨は初回だけ C。2回目以降は A または B。

## Step 3: 取得手段を選定

まず取得可否を確認し、以下の優先順位で使う。

| 手段 | 使う場面 | 注意 |
|---|---|---|
| WebSearch / WebFetch | 公式記事・紹介記事・公開URL候補の発見 | Threads本文の網羅取得には弱い |
| Threads公式API | 自アカウント投稿、インサイト、許可済みkeyword search | Meta App権限とトークン管理が必要。競合の全投稿取得には向かない |
| Playwright | ログイン済みWeb画面で検索・スクショ・DOM抽出 | Cookie管理と画面変更への耐性が課題 |
| Chrome / in-app browser系 | 既存ログイン・手動確認・視覚チェック | 自動で大量処理するならPlaywrightスクリプト化が必要 |
| 手動入力補助 | 認証やDOM取得が不安定な時 | 最初の30件調査では最も確実 |

### 初回チェック

1. 対象URLを未ログインのWebFetch/Playwrightで開く。
2. ログイン画面になる場合は、ログイン済みブラウザでの手動確認またはChrome系ツールを使う。
3. 取得できた場合も、スクショ・URL・投稿日・反応数を最低限保存する。

## Step 4: スプレッドシート構成

調査スプシには以下のタブを使う。なければ作成を提案し、ユーザー確認後に作る。

| タブ | 目的 |
|---|---|
| `調査設定` | 検索語、除外語、対象期間、最低反応数、優先媒体 |
| `発信者DB` | アカウントURL、bio、発信スタンス、商材導線、伴走度 |
| `投稿DB` | 投稿URL、本文、投稿日、反応数、分類、スクショURL |
| `構造分析` | フック、共感、具体例、CTA、改行、絵文字、文章密度 |
| `スクショDB` | スクショ保存先、撮影日時、対象URL、用途、確認状態 |
| `勝ちパターン仮説` | 伸びている型、真似る点、避ける点、根拠投稿 |
| `自アカウント転用案` | hiro/yuu向け投稿テンプレ、画像方針、ネタ帳候補 |

## Step 5: 分類ルール

### 発信者タイプ

- `伴走型`: 一緒に試す、つまずきを拾う、成長過程を見せる
- `講義型`: ノウハウを体系化して教える
- `実績型`: 数字・成果・権威を前面に出す
- `煽り型`: 危機感、置いていかれる不安、強い断定で動かす
- `日記型`: 日常・失敗・試行錯誤で親近感を作る
- `ツール配布型`: プロンプト、GPTs、テンプレ、チェックリストを配る

### 投稿構造タグ

- `フック強`: 1行目で悩み・意外性・数字・問いを置く
- `共感先行`: 解決策より先に読者の状態を言語化する
- `実験共有`: 自分で試した結果を出す
- `手順化`: 3ステップ、チェックリスト、テンプレ化
- `コメント誘導`: 質問、相談、合図ワードで返信を促す
- `DM誘導`: 無料相談、診断、配布物でDMへ送る
- `画像主導`: 画像・図解・スクショが主で本文は短い

## Step 6: 出力フォーマット

スプシ書き込み前に必ず以下を提示する。

```
調査下書き
- 対象テーマ: {theme}
- 候補発信者: {n}件
- 候補投稿: {n}件
- スクショ: {n}枚
- 書き込み先: {spreadsheetTitle} / {tabs}
- 副作用: 新規タブ作成 / 行追加 / Driveアップロード / なし

上記でスプシに反映してよいですか？
```

ユーザーが承認したら反映する。

## Step 7: 分析サマリー

調査後は以下を短くまとめる。

```
■ 伸びている見え方
- {例: 教えるより「一緒に試す」言い方が強い}

■ 投稿構造
- {例: 悩みの言語化 → 小さな実験 → 失敗も含めた学び → コメント誘導}

■ hiro_ai_dx に転用するなら
- {BtoB / AI導入 / 時短 / 診断導線}

■ yuu_workstyle に転用するなら
- {個人の成長 / 習慣化 / 等身大の実験 / 図解}
```

## 次のアクション提案

### 完了報告

✅ 調査モード完了
- 調査テーマ: {theme}
- 収集: 発信者 {creatorCount} 件 / 投稿 {postCount} 件 / スクショ {screenshotCount} 枚
- 副作用: {createdTabs / appendedRows / uploadedFiles / none}

### 次の選択肢（AskUserQuestion）

A. ネタ帳に追加する（mode-account-tend サブ③）
   — 採用した型を次回ループ用のネタにする

B. 投稿案を作る（mode-create-post）
   — 調査で見えた勝ちパターンを使って実投稿に変換する

C. アカウント設計を見直す（mode-account-tend サブ①/②/④）
   — ピラー・打ち分け・コンセプトに調査結果を反映する

D. 終了

### 選択時の動作

| 選択 | 動作 |
|---|---|
| A | Read `mode-account-tend.md` → サブ③ネタ帳追加 |
| B | Read `mode-create-post.md` → 調査結果をソースとして投稿作成 |
| C | Read `mode-account-tend.md` → ピラー/打ち分け/コンセプト更新 |
| D | 完了サマリーを短く再掲して終了 |
