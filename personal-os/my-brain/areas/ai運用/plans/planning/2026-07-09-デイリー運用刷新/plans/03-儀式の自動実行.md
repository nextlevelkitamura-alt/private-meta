> 1. **Orca起動の強化**: run.sh は Orca 不応答時にまず **Orca.app の起動を試み**（実測: `orca open` は「Orca 起動＋runtime到達待ち」・`/Applications/Orca.app` 実在）、応答するまで一定間隔でポーリング（上限既定60秒）。応答回復で可視ペイン spawn、上限超過で従来どおり headless フォールバック。可視ペインは Private repo で実行する（`DAILY_START_WT=name:Private`）。

## 目的

朝10:03に無人で「デイリースタート」を実行し、月週目標・昨日の引き継ぎ・繰越しを読んで、
今日の大課題（themes）と今日やること（todos）を**確定起票**する。人間の承認バーは置かない。
AIが気になった点だけ question としてボードに浮上させ、人間はスマホで回答するだけでよい。
手動で「デイリースタート」を実行した日は、10:03の定期実行は実行ログを見てスキップする。

## 運用モデル（確定・2026-07-20人間指示）

1. **朝ロックは自動**: 承認なしで themes/todos を確定起票。ドラフト状態（proposed）は使わない。
2. **気になる点だけ質問**: 重複・過剰・判断に迷う玉は `board.py ask` で question 発行（既存UI）。
3. **対話で修正可**: 起票後に人間がチャットで変更を指示すれば通常セッションが直す。
   最初は無人版で運用し、対話で決めたくなったら対話モードを既定へ切り替えればよい。
4. **朝の引き継ぎ例外**: 「繰越しは人間タップのみ」契約（focusmap todos.ts）に対し、
   daily-start の**朝の昨日→今日引き寄せだけは自動可**とする（2026-07-20人間指示）。
   未来方向への先送りは引き続き人間のみ。focusmap側コードは変更しない
   （daily-start は新todo起票＋carried_from 記録で引き継ぐ。既存todoのdo_dateは動かさない）。

## 実装スコープ（今夜）

### A. board.py `todo-add` サブコマンド（session-board）

- `turso/store.py` に `stmt_todo_insert` を追加、`board.py` に `todo-add` を追加（`goal-add`/`theme-add` と同型）。
- 引数: `--title`(必須) `--note` `--date`(既定=今日JST) `--due` `--repo`(既定 none) `--assignee`(self|ai・既定 self) `--route`(plan|routine|single・既定 plan) `--theme <theme_id>` `--carried-from <YYYY-MM-DD>` `--source`(既定 cli・CHECK内の値のみ)。
- id は uuid4。theme-add と同様に id を print（呼び出し元が紐付けに使う）。
- 既存テスト全PASS維持＋todo-add のテスト追加。MD・sessions系ロジックには触れない。

### B. Global Skill `daily-start`（skills/daily-start/）

- **無人モード**（`--auto`・loopから起動）:
  1. 冪等ガード: `loops-registry/loops/daily-start/state/done-<YYYY-MM-DD>` があれば即終了。
  2. 読む: 当月 `ゴール/デイリー/<年>/<月>/月間計画.md`（今月の的・週の大枠）／前日デイリーの「明日へ」節／
     繰越し候補（inbox todos: `do_date < today AND status='open'`）／前日 session_logs（文脈）。
  3. themes: 既存 active テーマを照合し、今日の大課題2〜3個を確定（不足分だけ `theme-add`・重複作成しない）。
  4. todos: 今日やることを `todo-add` で確定起票（テーマ紐付け・assignee/route 自己判定・**routineの自称は禁止**）。
     繰越しは新todo起票＋`--carried-from 昨日` で引き寄せる。
  5. 気になる点（過多・不明玉・衝突）だけ `board.py ask` で question 発行。
  6. 実行ログ `state/done-<YYYY-MM-DD>` を書き、`board.py log` で成果1行を記録して finish。
  7. デイリーmd本文の編集は行わない（8節構成は本programの他子の領分。壊さない）。
- **対話モード**（人間が「デイリースタート」と発話）: 同じ収集をして人間と対話で確定してから起票。
  無人版と同じ実行ログを書く（=手動実行日も10:03はスキップされる）。
- morning-routine との分担: daily-start=朝の起票（この子）。morning-routine の対話朝会・夜会は当面現状維持し、
  縮退・改名は人間判断待ちのまま（旧計画から変更なし）。

### C. loop `daily-start`（loops-registry/loops/daily-start/）

- `loop.md`（runner=ai・起動方式を明記）・`com.kitamura.daily-start.plist`
  （`StartCalendarInterval` 10:03 JST・`RunAtLoad` なし）・`scripts/run.sh`。
- `run.sh` は薄い起動役: flock → 当日 state 確認（あればスキップ・ログだけ残す）→
  Orca CLI 応答確認 → 可なら `cockpit.sh spawn` で可視1ペイン起動、不可なら `claude -p`（headless）
  フォールバック。モデルは `AIモデル一覧.md` のレーン規約に従い実装時に確定する。
- 実行ログ: `state/` に done マーカーと `output/` に各回ログ（loops-registry 既存流儀）。
- `実行loop一覧.md` へ1行追加・`verify.py` PASS・launchd load まで実施（人間承認済み）。
