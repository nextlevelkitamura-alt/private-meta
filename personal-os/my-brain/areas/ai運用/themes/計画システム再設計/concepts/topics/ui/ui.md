# Focusmap UI Topic

Focusmapの画面を、実装・DB・公開設定へ進む前に検討し、人間がPCとスマホの両方で判断できる状態にするためのTopicである。

## 正本

この `ui.md` は、UI検討におけるAIと開発者向けの唯一の正本である。画面ごとに次を記録する。

- 画面の目的と利用者が完了したいこと
- 表示する情報、その優先順位、fixtureで表す状態
- 操作点と「操作すると何が起きるか」
- スマホとPCで共通にするもの、変えるもの
- 採用した視覚方針と、その理由
- 採用モックの版と参照先

採用モックの参照先は、将来R2へ保存する場合もobject keyと版だけを記す。署名URL、公開URL、認証情報は記録しない。HTML本文・生成画像・実装コード・DB定義をこのファイルへ複製しない。

## モックの使い方

UIモックは、1枚の自己完結HTMLで作る。外部CSS、JavaScript、外部画像に依存せず、白背景で表示専用とする。同じシナリオ・同じfixtureを使い、PCではPCビューとスマホビューを並べ、狭い画面では縦に積んで比較できるようにする。

各モックには必ず次を含める。

- 画面名、版、表示状態、同時に比較するシナリオ
- この版で人間に判断してほしいこと
- PCとスマホの表示領域を示す見出し
- 操作点の注釈と、その結果
- 情報の優先順位または端末差の説明
- fixtureによる説明用表示であり、実データ・実装済み画面ではないこと

モックは意思決定を助ける表示物であり、画面仕様の正本ではない。採否、情報、操作、端末差の確定内容はこの `ui.md` に戻す。

## ローカル試作の置き場

画像生成案と意思決定前のHTMLは、Git対象外の `.local/` にだけ置く。画面名は実装名に縛られない短いslugを使う。

```text
.local/
  images/
    mobile/<screen>/
    desktop/<screen>/
  mocks/
    <screen>/
```

- `images/mobile/<screen>/`: スマホ向けの視覚方向案
- `images/desktop/<screen>/`: PC向けの視覚方向案
- `mocks/<screen>/`: 比較・判断用の試作HTML

採用前の画像と試作HTMLはコミットしない。採用時に必要な画像は、採用HTMLへ埋め込み、将来のR2保存用HTMLを自己完結にする。

## 既存html Skillとの境界

グローバル `html` Skillの白背景、自己完結、Markdown正本をHTMLで置き換えないという原則を使う。ただし、現時点ではSkillを変更しない。今回のFocusmap UI全体モックが完成し、複数Themeでも再利用できるレイアウト、注釈、PC/スマホ比較の完成形が固まった時だけ、その共通部分を既存html Skillのワークフローへ昇格する。
