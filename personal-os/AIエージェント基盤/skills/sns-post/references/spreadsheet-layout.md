# spreadsheet-layout — スプシ列レイアウト

両リポのスプシ「投稿管理」シートの共通列構造。

## 列定義

| 列 | 名称 | 内容 | 書き込みタイミング |
|---|---|---|---|
| A | 投稿予約 | TRUE/FALSE チェックボックス | 追加時=TRUE / 予約後=FALSE |
| B | 予約元 | "Buffer" / "個人API" 等 | Buffer登録後に "Buffer" |
| C | 画像URL | Drive公開URL | 画像生成後（モード3） |
| D | 投稿内容 | 本文 | 追加時 |
| E | 投稿日時 | YYYY-MM-DD HH:mm（JST） | 追加時 |
| F | 投稿先 | アカウント名 | 追加時 |
| G | ピラー | ピラー名（① 励まし系 等） | 追加時 |
| H | threads_id | Threads投稿ID | インサイト同期時（モード6） |
| I | 閲覧数 | views | インサイト取得時 |
| J | いいね | likes | インサイト取得時 |
| K | 返信数 | replies | インサイト取得時 |
| L | スコア | views×1 + likes×10 + replies×15 | インサイト取得時 |
| M | Buffer ID | Buffer 上の post ID | Buffer登録時 |

## A列・B列の状態遷移

| A列 | B列 | 意味 |
|---|---|---|
| TRUE | （空） | 投稿案完成・Buffer 未予約 → モード4で予約対象 |
| TRUE | Buffer | （通常はこの状態にならない） |
| FALSE | （空） | 下書き状態（投稿予定なし） |
| FALSE | Buffer | Buffer 予約済 / 投稿済 |

モード4（publish）で TRUE+空 → FALSE+Buffer に遷移させる。

## ヘッダー行（1行目）

```
A1: 投稿予約
B1: 予約元
C1: 画像
D1: 投稿内容
E1: 投稿日時
F1: 投稿先
G1: ピラー
H1: threads_id
I1: 閲覧数
J1: いいね
K1: 返信数
L1: スコア
M1: Buffer ID
```

## シート名

config の各 account の `sheetName` を参照。
- 仕事リポ: `kurashi_to_hataraku投稿管理` / `nextlevel__career投稿管理`（アカウント別）
- 副業リポ: `投稿管理`（単一・F列で投稿先を区別）

## 範囲指定の慣例

| 用途 | 範囲 |
|---|---|
| 投稿対象一覧 | A2:G500 |
| インサイト同期 | A2:M500 |
| 個別行の更新 | {列}{行番号}（例: F14） |
| アカウント管理シート | アカウント管理!A1:Z50 |

## アカウント管理シート

各リポのスプシに別タブ「アカウント管理」がある（Buffer の account-sync.ts で更新）：

```
A列: 項目名
B列以降: アカウントごとの値（B=アカウント1, C=アカウント2, ...）

行1: 空白
行2: アカウント名
行3: プロフィール画像（=IMAGE() 数式）
行4: 自己紹介文（手動入力）
行5: アカウントコンセプト（手動入力）
行6: 投稿打ち分け（手動入力）
```

## 調査タブ

`mode-research.md` で外部アカウント・投稿・スクショ・勝ちパターンを管理する。初期タブは以下。

### 調査設定

| 列 | 名称 | 内容 |
|---|---|---|
| A | 項目 | 検索語 / 除外語 / 対象期間 / 最低反応数 / 優先媒体 |
| B | 値 | 実際の設定値 |
| C | メモ | 運用メモ |

### 発信者DB

| 列 | 名称 | 内容 |

| A | 調査日 | YYYY-MM-DD |

| C | 表示名 | 発信者名 |
| D | アカウントID | @handle |
| E | URL | プロフィールURL |
| F | bio | 自己紹介 |
| G | 発信者タイプ | 伴走型 / 講義型 / 実績型 / 煽り型 / 日記型 / ツール配布型 |

| L | 避ける点 | 自アカウントに合わない点 |
