# 指揮官セッション(デスクトップチャット)の完了可視化

## 目的
デスクトップ指揮官(ccdチャット)の完了/停止を人間が気づけるようにする。watch.shの守備範囲はorcaペインのみであり、デスクトップチャットの完了は現在どこにも検知・通知されない。

## 現状
- watch.shはorcaペインのプロセス状態・マーカーファイルを監視するが、デスクトップのClaude Codeセッション(ccd)は対象外。
- 人間がデスクトップチャットの完了に気づかず放置する問題がユーザーから指摘されている。
- 受け渡し自動化(watch.sh WATCH_AUTO_REVIEW)はorcaペイン間の話であり、この問題は解決しない。
- インボックス原文(2026-07-03デイリー): 「指揮官セッション(デスクトップチャット)の完了可視化: watch.shの守備範囲はorcaペインのみで、デスクトップ指揮官の完了/停止に人間が気づけない。keeperのccdセッション巡回 or Claude Code Stopフック通知のどちらが軽いか調査→小さく実装」
- 出所: ユーザー指摘「指揮官の内容が終わっているの気づかない」(2026-07-03 17:40台)

## 方針
2案を調査し、軽い方を採用する:

1. **keeper の ccd セッション巡回**: keeper(見張り番)にデスクトップccdセッションの状態を定期チェックさせ、完了/停止を検知したら通知。
2. **Claude Code Stop フック通知**: Claude Codeのhook機構(PostToolCall等)でセッション終了時にmacOS通知やデイリー行注記を発火。

調査観点: 実装量・依存・誤検知リスク・既存機構との整合。
