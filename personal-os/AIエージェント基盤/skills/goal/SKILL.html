goal — 完遂実行ゲート

目的と完了条件が定まった仕事を「打てば done まで走らせる」実行コマンド。合意してから作る、を構造で強制し、合意後は人手の逐次指示なしに完遂まで運ぶ。

① 何をするSkillか

/goal <テーマ|計画> を起点に、入場条件の確認 → 計画の確保 → 直列自律実行 → [SAVE]評価 → done → テーマ更新を一直線に回す実行driver。

  • 入場条件で実装ファーストを止める: theme.md に「完成形スケッチ」と検証可能ゴールが無ければ実装に入らず壁打ち(grill-me)へ差し戻す（特に Codex 向けの制御）。
  • 合意後は自律: 実行ライン方式(テンプレv3)の計画を上から直列に走らせ、危険操作だけ人間ゲートで止める。
  • 規約・評価合否・close 判定は再定義せず、plan-registry / plan-ops / areas §1.2・§3 を呼ぶ。

② いつ発火するか

合意済みの目的を「実行フェーズ」へ渡す発話で呼ぶ。

goal / /goal これ完遂まで走らせて 一気通貫でやって 目的は決まったから実装まで回して

起票だけ = kickoff。計画の作成・合流・評価・終了の管理入口 = plan-create-review。goal はそれらを呼ぶ実行driver。

③ フロー（合意 → 完遂）

⑥ 絶対ルール・安全方針

止める条件

完成形スケッチ未確定のまま実装に入らない。planning のまま実装先行しない。git 未追跡の他者WIP を無断で移動・改変しない。GLOBAL_AGENTS §7 の危険操作は人間ゲート。

  • 正本は複製しない: テーマ層=areas/AGENTS.md §1.2、実行ライン/[SAVE]=§3・plan-registry §2、自律境界=GLOBAL_AGENTS §5–7、機械手続き=plan-ops。goal はこれらを参照するだけ。
  • 常駐しない: ユーザーが /goal で呼んだ時だけ動く。
  • 露出: 既定4窓（agents / claude / gemini config / gemini antigravity）＝全runtime。
