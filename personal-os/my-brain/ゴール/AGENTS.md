# ゴール（縦のladder）

3年→年間→デイリーの「的」と「履歴」を置く縦のladder。
横の領域別は `../areas/`。全体の「今」は当日デイリーの自動区画（今やっていること/待ち/計画ボード）で見る（`../ダッシュボード.md` はポインタ）。縦と横を混ぜない。

## 構成
- `3年計画.md`: 3年後にありたい状態（的）。
- `年間計画/<年>.md`: その年の的。3年計画から落とす。
- `デイリー/<年>/<月>/<年-月-日>.md`: 1日1枚・追記の履歴。
- `templates/デイリー.md`: 移設済みポインタ（雛形の正本=基盤repo `loops-registry/loops/renderer/templates/デイリー.md`）。
- `朝夜ルーティン.md`: ポインタのみ（毎日の運用手順の正本=Global Skill `morning-routine`。2026-07-03移行・逆算Alignmentは夜会手順に吸収）。
- `references/`: ゴール設定インタビュー等の参照テンプレ。

## ゴールが未設定/薄いとき
3年/年間が「未確定」だらけだと逆算も計画ボードも朝夜ルーティンも空回りする。`references/ゴール設定インタビュー.md` の問いに沿って対話で引き出し、固まった分だけ記入する（全部一度に埋めなくてよい）。

## ルール
1. デイリーは1日1枚。当日分は基盤のレンダラ（`loops-registry/loops/renderer/`）が自動生成する。人間はautoマーカーの外へ追記していく（上書きしない）。
2. デイリーには必ず「逆算チェック」を書く（今日やることが年間/3年目標に繋がるか）。
3. 進捗・実行状態は目標ファイルに書かない。状態は当日デイリーの自動区画（レンダラが描画）が持つ。
4. 目標の中身（実際の3年/年間目標）は人間が記入する。AIは雛形と置き場まで。
5. secret / token / 認証値は書かない。
