# 実行一覧

このマシンで動く **launchd 自動実行** の横断インデックス。実体（plist・スクリプト）の正本は各 repo。ここは「何が・いつ・何をしているか」を1画面で見るための索引。

## ファイル
- `<repo>.md` … repo ごとの一覧。現状 `personal-os.md` / `nextlevel-career.md`。
- 各 md の構成（上から）: **どのようなことを自動実行しているのか → 実行管理の仕組み → ジョブ → 実行メニュー**。

## セクションの意味
- **ジョブ** … 自動（launchd）で回っているもの。スケジュール付き。
- **実行メニュー** … 実行できるものの種類（手動オンデマンド含むカタログ）。ジョブはこの部分集合（自動化された一部）。

## 更新ルール
- launchd を追加・変更・停止したら: ①その repo の**正本**（あれば）を更新 → ②該当 `<repo>.md` の簡易行を更新する。
- **稼働状態（loaded/disabled）など変動値は中央 md に固定で書かない** → 各 md の「実態確認」コマンド（例: `status.sh` / `launchctl list`）で見る（md＝あるべき姿／コマンド＝実際）。
- **正本が別 repo にある**もの（例: nextlevel＝仕事マニュアル）は、中央 md を**簡易索引＋backlink**に留め、詳細を二重に持たない。**中央 md 自体が正本を兼ねる** repo（例: personal-os）はこの限りでない。

## スコープ外
- **hook は `../../hooks/` で管理**（ここには載せない）。cron は現状なし。
- ジョブ本体の実装・plist 正本は各 repo。ここは索引だけ。
