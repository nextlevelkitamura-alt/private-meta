# 実演例1: サクッと（合成例）

## 入力

- やりたいこと: 「`skills/inbox-triage/SKILL.md` の誤字（『行末に半角で』の後の句読点抜け）を直したい」
- 対象repo見当: AIエージェント基盤

## 4判断

1. **規模**: 問い1(1〜2ファイル)=YES（1ファイルのみ） / 問い2(容易に戻せる)=YES（1コミットrevertで戻る） / 問い3(人間ゲート事項を含まない)=YES（削除・push・main反映を含まない） → **サクッと**。
2. **経路**: 対象repo=AIエージェント基盤（本体）。計画置き場は不要（サクッとは計画書なしの即実行）。担当指揮官=既存のAIエージェント基盤指揮官チャットにそのまま合流。
3. **起動形**: サクッとなので構成カード自体を省略。指揮官直の編集、または単発ペイン1枚（実装役のみ）で足りる。
4. **モデル**: 既定sonnet5。このレーンが遅くても他に待つレーン・人間はいない → 格上げなし。

## 出力

構成カードは省略（契約§2の定義どおり）。指揮官が直接修正、または単発ペインで実装後、事後報告のみ:

```
サクッと処理: skills/inbox-triage/SKILL.md の誤字修正（1ファイル・revert容易・人間ゲート無し）。
実行: 指揮官直 / モデル: sonnet5（既定）。完了。
```
