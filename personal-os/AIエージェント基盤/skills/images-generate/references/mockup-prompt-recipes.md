# Mockup Prompt Recipes

Use these recipes only when a task needs more specific prompt structure than `SKILL.md`.

## LP Hero Mockup

```text
Use case: ui-mockup
Mockup type: lp-hero-mockup
Asset type: landing page hero image
Product: <name>, <one-line product purpose>
Target feature/screen: <primary feature shown>
Audience: prospective users deciding whether the product solves their workflow
Fidelity: polished future concept grounded in the current product
Style: refined SaaS product visual, calm operational interface, realistic browser or device surface
Composition: wide hero-ready composition with the product as the first visual signal, enough negative space for headline copy if needed, next-section hint possible
UI content: short English labels only, such as "<label 1>", "<label 2>", "<label 3>"
Brand/context cues: align colors, icon style, density, rounded corners, and state colors with existing product UI
Constraints: no unrelated logos, no fake brand, no decorative abstract dashboard, no unreadable tiny text, no watermark
Avoid: stock-photo feel, random analytics charts, excessive glow, generic AI motifs
```

## Device Product Mockup

```text
Use case: product-mockup
Mockup type: device-product-mockup
Asset type: product presentation image
Product: <name>, <one-line product purpose>
Target feature/screen: <screen or workflow shown>
Audience: users who need to quickly understand the product experience
Fidelity: polished but plausible UI concept
Style: premium product photography of devices with realistic screens
Composition: <phone/laptop/tablet/desktop> device frame, screen content clearly visible, simple background, product UI is the focal point
UI content: short English labels, realistic status chips, compact navigation
Brand/context cues: use product colors and UI density; preserve existing app identity
Constraints: no fake app store badges unless requested, no unrelated operating system branding, no illegible microtext, no watermark
Avoid: floating cards unrelated to the screen, random charts, over-stylized neon sci-fi
```

## Pitch Or Investor Demo Visual

```text
Use case: productivity-visual
Mockup type: pitch-demo-visual
Asset type: pitch deck product concept image
Product: <name>, <one-line product purpose>
Target feature/screen: <high-value workflow or outcome>
Audience: investors, stakeholders, or partners
Fidelity: future polished concept that is still believable from the current product
Style: boardroom-ready product visualization, clear hierarchy, strategic and credible
Composition: one central product surface with supporting workflow context, show before/after state or approval loop if useful
UI content: short English labels that explain value, not long marketing copy
Brand/context cues: match current product visual language; emphasize operational control and measurable progress
Constraints: make the product understandable without a long caption; avoid fake metrics unless clearly illustrative
Avoid: generic startup pitch art, humanoid robots, abstract AI clouds, impossible UI density
```

## Feature Explainer

```text
Use case: infographic-diagram
Mockup type: feature-explainer
Asset type: feature explanation image
Product: <name>, <one-line product purpose>
Target feature/screen: <feature being explained>
Audience: users learning what the feature does
Fidelity: clean product diagram with realistic UI fragments
Style: structured explainer, lightweight diagram plus product UI, minimal text
Composition: left-to-right or top-to-bottom flow with 3-4 steps maximum
UI content: short English labels for each step, exact required words in quotes
Brand/context cues: use product colors, line weights, state chips, and icon style
Constraints: keep steps visually distinct; no dense paragraphs; no arbitrary features outside the request
Avoid: complicated flowcharts, decorative filler, generic process diagrams
```

## Before/After Concept

```text
Use case: productivity-visual
Mockup type: before-after-concept
Asset type: transformation visual
Product: <name>, <one-line product purpose>
Target feature/screen: <workflow transformation>
Audience: users who need to understand the improvement
Fidelity: concept visual with product UI fragments
Style: restrained split comparison, practical and clear
Composition: two panels labeled with short English captions, before state visibly fragmented, after state organized through the product
UI content: "Before", "After", plus 2-3 short labels maximum
Brand/context cues: make the after panel clearly match the product identity
Constraints: do not exaggerate with fake metrics; keep the comparison credible
Avoid: chaotic unreadable before panel, dramatic but meaningless visuals, unrelated tools
```
