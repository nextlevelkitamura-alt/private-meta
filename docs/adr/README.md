# Architecture Decision Records

Use this directory for decisions that affect architecture, governance, compatibility, or long-term maintenance.

Suggested file name:

```text
ADR-001-short-title.md
```

Suggested sections:

- Status: proposed, accepted, deprecated, or superseded.
- Context.
- Decision.
- Consequences.
- Alternatives considered.
- Related requirements.

Do not use ADRs to hide unresolved product requirements. Link unresolved items back to `docs/requirements/contradictions.md`.
