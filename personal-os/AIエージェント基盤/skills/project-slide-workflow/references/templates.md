# テンプレ選択・保存ルール

## 目的

スライド作成時に毎回UIをゼロから決めないため、使うロゴ・色・余白・カード・図解の型を先に選ぶ。  
完成後に良いUIができたら、次回使えるテンプレとして記録する。

## テンプレ選択フロー

資料作成の初期に、必要なら次のように聞く。

```markdown
今回のUIはどれで進めますか？

1. NEXTLEVEL CAREER 白×赤ミニマル
2. 既存スライドの見た目を踏襲
3. 今回だけ新規で作る
```

ユーザーが「前回の感じ」「さっきのUI」と言った場合は、直近で採用されたスライド画像をUI参照として使う。

## 標準テンプレ: NEXTLEVEL CAREER 白×赤ミニマル

用途:
- キャリア支援プロジェクト
- 求人・派遣・就業サポート説明
- 参加者向け資料
- LINE/LP/Instagramに転用する説明スライド

ロゴ:
`/Users/kitamuranaohiro/マイドライブ（nextlevel.kitamura@gmail.com）/01_ネクストレベル/素材/ネクレベ　キャリア　ロゴ/ネクストレベル　キャリア　ロゴ　横長.jpg`

UI指定:
- 16:9
- 白背景
- 余白多め
- 黒または濃いグレー本文
- 赤をアクセントに使う
- 表紙はロゴ大きめ
- 2枚目以降はロゴ控えめ
- 薄いグレーの波形・線・カード背景は可
- 丸みのある薄いカード
- 線画アイコン
- ロードマップは横並びで、赤の細い線か矢印でつなぐ

避けるもの:
- 上部の不要なヘッダー
- CTAボタンの乱用
- 濃い背景
- 派手なグラデーション
- 写真素材感の強い人物写真
- 文言の自動追加

文体:
- 「一緒に整理します」
- 「一緒に進みます」
- 「相談できます」
- 「できる場合があります」
- 「条件が合えば」

## 完成後テンプレ化

スライド作成完了後、次を確認する。

```markdown
今回のUIを次回用テンプレとして残しますか？
残す場合は、テンプレ名をつけて、ロゴ・色・レイアウト・採用画像・プロンプトを記録します。
```

保存する場合に記録する項目:

- テンプレ名
- 用途
- ロゴパス
- 色
- 文字量の目安
- ページ構成
- 採用画像パス
- 共通画像生成プロンプト
- 使う時の注意

保存先の目安:
- グローバルSkill内: `references/templates.md` に追記
- 業務リポで管理する場合: `領域/整備/スキル整備/` など、ユーザーが指定した場所
