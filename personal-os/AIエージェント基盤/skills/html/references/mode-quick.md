# mode-quick（軽い・低トークン・使い捨て）

その場の説明、簡単なレポート、今の計画をぱっと見せる等、一度見て終わるものに使う。速さと軽さを優先する。

## 方針

1. 構造は固定しない（自由でよい）。最小限の型だけ:
   - 見出し ＋ 1行結論
   - 要点（箇条書き、または小さいカード）
   - 詳細を書くなら、節の型（リード1行 → 主役部品1つ）だけは quick でも守る
2. 部品の選び方は full と同じ `html-structure.md` §3 の選択ラダー（1段目は図解判定）。構造・流れ・関係が主題なら、quickでも `diagram-recipes.md` の雛形（R1〜R6）ベースの図を既定で1〜2枚入れる。凝った描き起こし（多段レーン・大型関係図）はfullだけ。装飾目的だけの図は禁止。予算の軽さは「使う部品は2〜3種まで・CSSは使う部品の分だけ・図は雛形コピペに限定」で守る。
3. 表を使うなら「小さい表」＝ 5行×3列以内、かつ `html-structure.md` §4 の5条件を満たす時だけ。超える詳細は `.deflist` か箇条書きへ。
4. 低トークン: 色は控えめ（意味のある箇所だけ）。
5. 軽量化しても白背景のライト単色は省略しない。暗い背景と表示環境による配色切替は作らない。
6. 使い捨て前提。残す / 繰り返し見る / メタ的に重要、なら `mode-full.md` に切り替える。

## 迷ったらこれ

1. 「後でまた見るか？」がNo → quick。
2. 30秒で読めて用が済むもの → quick。
