# Output Schema

The canonical transcript JSON is validated conceptually by this skill's `schemas/transcript.schema.json`.

## Source

- `source.input_path`: original video/audio path.
- `source.audio_path`: extracted or direct audio path passed to Whisper.
- `source.duration_sec`: audio duration from `ffprobe` when available.
- `source.language`: transcription language code.

## Engine

- `engine.name`: `mlx-whisper`.
- `engine.model`: model repo or local model path.
- `engine.word_timestamps`: must be `true` for editing-grade output.
- `engine.condition_on_previous_text`: usually `false` for long-form hallucination resistance.

## Segments

Segments are Whisper's larger chunks. They are useful for review, but they are not precise enough to be the editing source of truth.

Required fields:

- `id`
- `start`
- `end`
- `raw_text`
- `unit_ids`

## Units

Units are built from `segments[].words[]`. They are the source of truth for edit timestamps.

Required fields:

- `id`
- `segment_id`
- `start`
- `end`
- `text`
- `probability` when available

Do not create one-character timestamps.

## Caption Blocks

`caption_blocks` are generated by post-processing `units`.

Required fields:

- `id`
- `start`: copied from the first unit.
- `end`: copied from the last unit.
- `unit_ids`
- `raw_text`
- `clean_text`
- `line_breaks`
- `duration_sec`

LLMs may update `clean_text` only if meaning and timing are preserved. LLMs must not generate or modify `start` / `end`.
