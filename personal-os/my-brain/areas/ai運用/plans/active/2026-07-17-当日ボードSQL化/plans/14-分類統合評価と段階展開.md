# 分類統合評価と段階展開

## 目的

CodexとClaudeのセッション受付からFocusmapの人間確定・単発完了までを統合評価し、安全に段階展開する。

## 非対象

- 評価未完了のまま自動確定範囲を広げること
- 未分類を隠すこと

## 現状

子10〜13の統合評価を所有する末端子。各子の単体検証だけではruntime差・DB障害・UI訂正を証明できない。

## 実行契約

- 対象repo: AIエージェント基盤＋focusmap
- 実行形: integration
- 最初に読む順番:
  1. AIエージェント基盤とfocusmapの最寄りAGENTS.md
  2. ../program.md
  3. ../実装/共通.md
  4. この計画
  5. 子10〜13の実装結果・評価文書
- 依存成果: 子10〜13の実装・評価結果
- 変更可能範囲: 評価文書、検証用の既存test、合意HTMLの更新
- 変更禁止範囲: 評価未承認の本番自動確定拡大、既存データ削除
- ファイル担当マップ: 不要
- worktree方針: task-scoped（focusmap評価時）
- 維持する契約: DB障害でも対話を止めず、判定待ちを消さず、曖昧一致は人間確認へ残す
- 検証: Codex/Claude実セッション、未登録Git、worktree、非Git、同名repo、DB断、再開/compact、subagent、採用/却下/完了
- 停止・エスカレーション条件: 誤分類率・未記録率を測れない、Codex/Claudeで意味差が残る、対象DBまたはrollbackが解決できない
- 完了時に返す情報: result packet（status / base_commit / result_commit / changed_paths / tests / assumptions / blockers / remaining_risks / out_of_scope_findings）
