# 内容すり合わせ

ユーザーの依頼をすぐ実行せず、まず内容をすり合わせてください。

目的は、ユーザーが求めている内容を高解像度に言語化し、実行前の認識ズレをなくすことです。

## 出力

以下を短く整理してください。

1. 理解した内容
2. 目指すゴール
3. 曖昧な点
4. 確認したい質問 最大3つ
5. 仮に進めるならどうするか

## ルール

- まだ実装、編集、詳細計画、調査実行はしない
- 質問は最大3つまでにする
- 曖昧な場合は、可能ならA/B/Cの選択肢で聞く
- 十分に明確なら「この理解で進めてよいですか？」で締める
- ユーザーの入力が短くても、勝手に大きな仕様へ広げない
