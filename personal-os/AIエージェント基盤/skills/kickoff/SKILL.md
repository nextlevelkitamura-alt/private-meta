# kickoff（起票ゲート）

「何かをやる」の入口を1つに揃え、**出所が必ずデイリーに残る**ようにするゲート。緊急直依頼で「なぜ・誰の指示で動いているか」が消える穴の根治（マルチ指揮官体制program 2026-07-03午後裁定・子05）。

正本ポインタ（本文に複製しない）:

- 規模・段階語彙・人間ゲート: `~/Private/personal-os/AIエージェント基盤/GLOBAL_AGENTS.md` §7
- 判定基準（規模/経路/起動形/モデル）: `plan-registry`（規約＋経路解決＝triage決定手続き §6）
- 起案結果マーカー（→計画作成済み／→サクッと判定）: skill `inbox-triage`。着手・完了注記は本Skillの手順3〜4。
- 現在の担当・采配経路: 当日デイリーと session-board

## 手順（軽さ最優先・サクッとは起票→GOまで数十秒）

1. **1行起票**: 当日デイリー（`my-brain/ゴール/デイリー/<年>/<月>/<日>.md`）の `## 依頼インボックス` 節末尾へ追記する。形式: `- <やりたいこと1行>（起票: <誰/どのチャット> HH:mm）`。既存行の削除・書換はしない（追記のみ・auto:*区画に触らない）。
2. **軽量判定**: plan-registryの構造3条件だけを当てる（①変更1〜2ファイル ②容易に戻せる ③人間ゲート事項なし）。**追加の質問・計画書生成・調査はゲート内でしない**。迷ったらライト側に上げてゲートを出る（ゲート内で悩まない）。
3. **サクッと** → そのまま実行GO。起票行末に `→着手(担当X)` を追記し、完了時に `→完了` を追記。実行者は呼び出し元（指揮官直・ペイン直）でよい。
4. **ライト以上** → `plan-registry` の経路解決（triage決定手続き §6・`triage.md`）へ渡す（構成カード＋計画スケルトン→全体管理者の采配経路）。起案が済んだら起票行末に `→計画作成済み(<パス>)` を追記。計画は planning に置き、レーン起動を決めた全体管理者/人間だけが plan-ops の `bucketctl.sh promote <計画フォルダ> --to active --apply`（定型コミットまで行うなら `--commit`）で昇格する。上限超過時の追い出しは自動化しない。

## AI側の自動通過（人間は明示呼び出し・AIは手順で必ず通る）

- 指揮官・ワーカーの手順正本（cockpit-supervisor／orca-cockpit references/role-prompts.md／morning-routine）に「作業開始=kickoff経由（1行起票してから動く）」を1行組み込む。hookでの全自動発火はしない（遅延回避・2026-07-03裁定）。
- 緊急でorcaペインへ直接依頼した場合も、着手した側が事後にこのゲートで1行起票する（全体管理者の代筆可）。

## 例外入口

Notion依頼インボックスからの自動pullは現在提供しない。外出時などにNotionへ書いた依頼は、人間が当日デイリーへ転記してから本Skillの手順1へ流す。
