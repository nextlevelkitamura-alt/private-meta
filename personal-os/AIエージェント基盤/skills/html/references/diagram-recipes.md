# diagram-recipes（図解レシピ集）

「この内容ならこの図」を機械的に引くための対応表。図解はhtmlスキルの既定の主役部品であり、テキスト部品は図で描けない内容だけの受け皿にする（判定の位置は `html-structure.md` §3の1段目）。SVG雛形の実体は `assets/artifact-template.html` の「図解レシピ雛形」ブロック（R1〜R6）だけに置き、ここへコードを複製しない。

## 1. 対応表（主題の形→レシピ）

1. フォルダ構成・正本とruntime露出・包含関係 → **R1 ボックス型ツリー図**。正本は塗り、symlink・露出先は破線。テキストの `.tree` は details内の補足・全量列挙用。
2. 手順・パイプライン・発火の流れ → **R2 横フロー図**。人間ゲート・承認点は琥珀（warn）で強調。分岐・状態遷移は菱形を足す。
3. 依存・参照・連携・責務の関係 → **R3 関係図**。矢印の向き＝参照の向きに固定し、線の近くに動詞ラベルを置く。
4. 層・積み重ね（runtime／基盤／正本など） → **R4 レイヤー図**。今回触る層だけ着色する。
5. 変更前後の構造・設定の差 → **R5 before/after対比図**。同じ図法で並べ、差分だけ緑（go）で着色。
6. 経緯・時系列・現在地 → **R6 タイムライン**。現在地を塗りで示し、細かい経緯は details へ。

どれにも当たらない（列挙可能な事実・長い理由・逐条規約が主題）→ テキスト部品ラダー（`html-structure.md` §3の2段目以降）へ。

## 2. 量の目安（縛りすぎない）

1. **既定**: 各節でまず「この内容は上の対応表で描けるか」を判定する。当たれば図が主役部品。
2. **目安**: 主要節の過半で図が主役、1ページ最低1枚。厳格な比率ルールにはしない。
3. **例外（明記された逃げ道）**: 長い理由・逐条規約・列挙可能な事実はテキスト部品でよい。**装飾目的だけの図は逆に禁止**（図解は情報を持つときだけ）。
4. quickは雛形R1〜R6のコピペベースで1〜2枚に留める。凝った描き起こし（多段レーン・大型関係図）はfullだけ。

## 3. 図の中身の規約

1. 色はテンプレのCSSトークンのみ。白背景ライト単色で、強調は1色（accent）、推奨・完了・差分=go、注意・人間ゲート=warn。
2. 実線＋塗り＝正本・確定、破線＝symlink・未確定・変更禁止範囲。凡例が要る区別は figcaption に1行で書く。
3. 図中テキストは名詞句・目安15字以内。説明文は図に書かず figcaption か本文リードへ。
4. `viewBox` で描き `width=100%`。スマホは figure の横スクロールで守る（テンプレ既定）。
5. 1枚に詰めすぎない。ノードが10個を超えたら節を分けるか、周辺を details のテキストへ落とす。
6. 必ず `role="img"` と `aria-label` を付ける。

## 4. 使い方

1. `assets/artifact-template.html` の「図解レシピ雛形」ブロックから該当レシピをコピーし、ノード数・ラベル・座標を内容に合わせて調整する。
2. 図＋figcaption を節の主役部品として置く（節の型は `html-structure.md` §5）。
3. メタ説明での使い分けは `meta-explain-layout.md` §3 がこの対応表を参照する。
