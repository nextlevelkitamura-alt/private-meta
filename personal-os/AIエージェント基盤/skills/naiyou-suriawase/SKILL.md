# 内容すり合わせ

実装や変更を始める前に、AIと人間の認識の相違をなくす。
基本的な実装依頼なら、明示されなくても着手前に毎回使う。

## 出力（この2つだけ）

**目指すゴール**

- 依頼が達成された状態を1〜2行で書く。
- UI実装やフォルダ構成の変更を含むときは、ゴールの下に変更前→変更後の簡易テキスト図（フォルダツリー・画面ラフ）を出す。

**確認したいこと**

- 最大3つ。**質問文そのものを太字にして強調する。**
- 可能ならA/Bの選択肢で聞く。

## 出力例

> **目指すゴール**
>
> 設定画面にダークモード切替を追加する
>
> ```
> ┌─ 設定 ──────────────┐
> │ 通知         [ON ]  │
> │ ダークモード [OFF]  ← 追加
> └─────────────────────┘
> ```
>
> **確認したいこと**
>
> 1. **切替は即時反映ですか？（A: 即時 / B: 再起動後）**

## ルール

- 回答をもらうまで実装・編集をしない
- 「理解した内容」「曖昧な点」「仮に進めるなら」などの節は書かない
- 図はどのruntimeのチャットでも表示できるテキスト図にする
- 依頼が短くても勝手に仕様を広げない
