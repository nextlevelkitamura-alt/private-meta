# チェックボックスとパーサ追従

## 目的

子計画マップの各行左に `- [ ]`/`- [x]` を付けて進捗を可視化し、progctl と program-lint がそれを正しく解釈するようにする。

## 現状

`_planops_map.py` の `HEADER_RE = ^(\d{2})...`(行頭2桁数字)でブロックを検出する。左に `- [ ]` を付けると当たらず、ブロック0件検出になる。結果:
- `progctl.sh` → 「NNブロック無し」で**停止**(気づける)。
- `program-lint.sh` → 検査0件で**「違反なし」と誤合格**(気づけない=危険)。
既存 active に 6 つの program がある(移行対象)。

## 方針

1. `_planops_map.py` の HEADER_RE を `^(?:- \[[ x]\]\s+)?(\d{2})...` へ拡張し、チェックボックス有無どちらの見出しも解釈する。
2. `progctl_core.py` の header 書換がチェックボックスを保持する。`program_lint_core.py` に「`[x]` ⟺ 状態=完了」の整合チェックを追加する。
3. `__tests__/fixtures` と 4 テストを新フォーマットで更新。既存 6 program の子計画マップを一括移行する。
4. 状態=完了に揃ったら progctl が該当行を `[x]`、それ以外は `[ ]` に同時更新する(状態と箱の二重管理を避ける)。

## 完了条件（レビュー項目）

- [ ] `_planops_map.py` が `- [ ]`/`- [x]` 付き・無し両方の見出しを検出する(テスト有り)。
- [ ] `program_lint_core.py` が `[x]` と 状態=完了 の不整合を検出する(テスト有り)。
- [ ] `progctl.sh set --state 完了` が該当行を `[x]` に同期する。
- [ ] 既存6 program が新フォーマットへ移行され、`program-lint.sh` が0件検出の誤合格でなく実チェックを通す。
- [ ] `__tests__/run.sh` が全て緑。
