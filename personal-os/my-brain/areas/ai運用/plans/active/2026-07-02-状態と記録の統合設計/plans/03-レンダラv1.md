# 03 レンダラv1（統合デイリー生成・pull取込）

## 目的

当日デイリー1枚を唯一の「見る・書く」面にするレンダラを基盤repoに実装する（設計正本: `../program.md` 方針§1/§3/§4、運用契約§3）。

## 方針（実装スコープ）

1. 当日ファイル自動生成: 新テンプレ（8セクション: 逆算 / 今日のTODO / 依頼インボックス / 今やっていること / 待ち / 計画ボード / 今日終わったこと / ログ。auto:carry・auto:progressは置かない）から生成。auto:goalは年間計画から決定的に転記。人間行・マーカー外は不可侵。
2. `auto:*` 冪等再生成: auto:done（Claude transcript＋git件名解決＋Codex分）／ auto:align（件数）／ auto:log（既存hook形式を踏襲）。
3. Codex pull: `~/.codex/session_index.jsonl` から当日分（時刻・cwd/repo・要旨1行）を取り込む。hookは追加しない。
4. トリガ: 既存のClaude Stop hookチェーンから起動（async・60秒debounce）。23:30 の daily-digest は締めの最終レンダに変更。launchdの新規追加なし。
5. 実装場所: 基盤 `loops-registry/loops/renderer/`（daily-digestとの統合/置換は実装レビューで判断）。
6. 「実行中・待ち・計画ボード」区画の結線は子04スコープ（本子はデイリー基本区画まで）。

## 完了条件（レビュー項目）

- [ ] 当日デイリーが無ければテンプレから自動生成される（既存ファイルの人間行を壊さないテスト付き）
- [ ] Codexの実セッションが当日デイリーに載る
- [ ] auto:done が起動の度に再生成され、2回連続実行で差分ゼロ（冪等）
- [ ] 空の auto:carry / auto:progress がテンプレ・当日ファイルに無い
- [ ] hook/launchd の変更はドラフト提示のみで、適用は人間ゲート（運用契約§2/§7）
- [ ] secret・認証値を読まない・書かない
